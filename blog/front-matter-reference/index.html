Front matter reference

June 19, 2025 · Robert

Complete reference guide for YAML front matter in Stattic. Front matter is the metadata that appears at the top of your Markdown files, controlling how your content is processed, displayed, and organized.

Overview

Front matter is YAML metadata placed at the beginning of Markdown files, enclosed by triple dashes (---). It provides structured data about your content that Stattic uses for:

  • Page generation and URL structure
  • Content organization with categories and tags
  • SEO optimization with meta tags
  • Template selection and display options
  • Custom functionality through additional fields

Basic Structure

---
# Required fields
title: "Your Content Title"
date: "2024-01-15"

# Optional fields
description: "Brief description of the content"
tags: ["web", "tutorial"]
author: 1

# Custom fields
reading_time: 10
---

# Your content starts here

Your Markdown content follows the front matter...

Basic Syntax

YAML Formatting Rules

---
# Strings (can be quoted or unquoted)
title: "My Blog Post"
title: My Blog Post              # Also valid

# Dates (ISO format recommended)
date: "2024-01-15"
date: 2024-01-15                 # Also valid

# Numbers
reading_time: 10
order: 1

# Booleans
featured: true
draft: false

# Lists/Arrays
tags: ["web", "tutorial", "beginner"]
tags:                            # Alternative list syntax
  - web
  - tutorial
  - beginner

# Objects/Dictionaries
author:
  name: "John Doe"
  email: "john@example.com"

# Comments (ignored by parser)
# This is a comment
---

Common Syntax Errors

---
# ❌ Common mistakes
title: My Post's Title           # Unescaped apostrophe
date: 01/15/2024                # Wrong date format
tags: web, tutorial             # Should be a list
author: John Doe's Blog         # Unescaped apostrophe

# ✅ Correct versions
title: "My Post's Title"
date: "2024-01-15"
tags: ["web", "tutorial"]
author: "John Doe's Blog"
---

Required Fields

Title (Required)

The title of your content, used in HTML <title> tags, headings, and navigation.

---
title: "Complete Guide to Static Sites"
---

Usage:

  • Appears as the main heading
  • Used in browser tab titles
  • Displayed in post listings
  • Included in RSS feeds

Best Practices:

  • Keep titles under 60 characters for SEO
  • Make them descriptive and compelling
  • Use proper capitalization
  • Avoid special characters that break URLs

Date (Required for Posts)

Publication date in ISO format (YYYY-MM-DD).

---
date: "2024-01-15"
date: "2024-01-15T10:30:00"     # With time
date: "2024-01-15T10:30:00Z"    # With timezone
---

Usage:

  • Controls post sorting and pagination
  • Displayed in post metadata
  • Used for archive organization
  • Included in RSS feed timestamps

Formats Supported:

  • "2024-01-15" - Date only
  • "2024-01-15T10:30:00" - Date and time
  • "2024-01-15T10:30:00Z" - With UTC timezone
  • "2024-01-15T10:30:00-05:00" - With timezone offset

Optional Fields

Description

Meta description for SEO and social sharing.

---
description: "Learn how to build fast, secure static websites with this comprehensive guide covering tools, best practices, and deployment strategies."
---

Usage:

  • Meta description tag
  • Social media previews
  • RSS feed descriptions
  • Post excerpts (if no explicit excerpt)

Best Practices:

  • 120-160 characters for optimal SEO
  • Include target keywords naturally
  • Write compelling copy that encourages clicks
  • Avoid duplicating the title

Author

Reference to author information from authors.yml.

---
author: 1                       # Author ID
author: "John Doe"             # Author name (if no authors.yml)
---

Author ID Reference:

# In content/authors.yml
1:
  name: "John Doe"
  bio: "Full-stack developer"
  email: "john@example.com"
  avatar: "/images/authors/john.jpg"
  website: "https://johndoe.dev"

Custom URL

Override the default URL structure.

---
custom_url: "/guides/static-sites/"
custom_url: "/special-page/"
custom_url: "/blog/my-custom-slug/"
---

Default URL vs Custom:

  • Default: /posts/my-blog-post/ (based on filename)
  • Custom: /guides/static-sites/ (from custom_url)

Best Practices:

  • Start and end with forward slashes
  • Use hyphens instead of spaces
  • Keep URLs short and descriptive
  • Avoid changing URLs after publication

Content Organization

Categories

Organize content with categories (uses numeric IDs from categories.yml).

---
categories: [1, 2]             # Array of category IDs
categories:                    # Alternative array syntax
  - 1
  - 2
---

Category Definition:

# In content/categories.yml
1:
  name: "Tutorials"
  description: "Step-by-step guides and how-tos"
  slug: "tutorials"

2:
  name: "Reviews"
  description: "Product and service reviews"
  slug: "reviews"

Tags

Multiple tags for detailed content classification (uses numeric IDs from tags.yml).

---
tags: [1, 2, 3]               # Array of tag IDs
tags:                         # Alternative array syntax
  - 1
  - 2
  - 3
---

Tag Definition:

# In content/tags.yml
1:
  name: "Getting Started"
  description: "Posts about getting started with new topics"
  slug: "getting-started"

2:
  name: "Web Development"
  description: "Posts about web development"
  slug: "web-development"

Tag Best Practices:

  • Use 3-7 tags per post
  • Keep tags consistent across posts
  • Define all tags in tags.yml with numeric IDs
  • Use descriptive names and slugs

Featured Image

Specify a featured image for posts and pages.

---
featured_image: "images/posts/my-post-hero.jpg"
featured_image: "assets/images/hero.jpg"
---

Usage:

  • Displayed at the top of posts/pages
  • Used in social media previews (Open Graph)
  • Referenced in templates as page.featured_image

Best Practices:

  • Use relative paths from the output directory
  • Optimize images for web (WebP when possible)
  • Include alt text context in surrounding content

SEO and Metadata

SEO Metadata

Core SEO fields that Stattic actually uses.

---
# SEO Title (optional - defaults to title)
seo_title: "Custom SEO Title for Search Engines"

# Meta description
description: "SEO-optimized description for search engines and social media"

# Keywords for meta tags
keywords: "static sites, web development, JAMstack, performance"

# Language code
lang: "en"
---

Display Options

Custom Excerpt

Override the auto-generated excerpt.

---
excerpt: "Custom excerpt that overrides auto-generated one from content"
---

Template Selection

Choose custom templates for specific content.

---
template: "featured"           # Uses post-featured.html or page-featured.html
template: "landing"            # Uses page-landing.html
template: "tutorial"           # Uses post-tutorial.html
---

Page Ordering

Control the order of pages in navigation (if implemented in templates).

---
order: 1                       # Numeric ordering for pages
---

Custom Fields

Stattic passes all front matter fields to templates through the metadata and page variables, so you can add any custom fields you need and access them in your templates.

Accessing Custom Fields in Templates

---
title: "My Post"
reading_time: 15
difficulty: "advanced"
my_custom_field: "custom value"
---

In your templates:

<!-- Access via metadata -->
{{ metadata.reading_time }}
{{ metadata.difficulty }}
{{ metadata.my_custom_field }}

<!-- Or via page -->
{{ page.reading_time }}
{{ page.difficulty }}
{{ page.my_custom_field }}

Common Custom Field Patterns

---
# Content metadata
reading_time: 15               # Minutes to read
difficulty: "advanced"         # beginner, intermediate, advanced
last_updated: "2024-01-20"    # When content was last revised

# Display options
show_toc: true                 # Show table of contents
sidebar: false                 # Hide sidebar
full_width: true              # Use full width layout

# Custom categorization
topic: "web-development"       # Additional categorization
format: "tutorial"            # Content format type
---

Advanced Usage

Accessing Fields in Templates

All front matter fields are available in Jinja2 templates:

<!-- Basic fields -->
<h1>{{ title }}</h1>
<p>By {{ get_author_name(author) }}</p>
<time>{{ date }}</time>

<!-- Custom fields -->
{% if metadata.reading_time %}
<p>Reading time: {{ metadata.reading_time }} minutes</p>
{% endif %}

{% if page.difficulty %}
<span class="difficulty">{{ page.difficulty }}</span>
{% endif %}

<!-- Featured image -->
{% if featured_image %}
<img src="{{ featured_image }}" alt="Featured image for {{ title }}">
{% endif %}

Template-Specific Fields

You can create fields that are only meaningful for certain templates:

---
title: "Product Review"
template: "review"

# These fields only matter for review template
product_name: "Static Site Generator X"
rating: 4.5
affiliate_link: "https://example.com/product"
pros: ["Fast", "Easy to use", "Great documentation"]
cons: ["Limited themes", "Learning curve"]
---

Validation and Troubleshooting

Common Errors

YAML Syntax Errors

---
# ❌ Syntax errors
title: My Post's Title           # Unescaped apostrophe
date: 01/15/2024                # Wrong date format
tags: web, tutorial             # Missing brackets
author: 1,                      # Trailing comma

# ✅ Correct syntax
title: "My Post's Title"
date: "2024-01-15"
tags: ["web", "tutorial"]
author: 1
---

Data Type Errors

---
# ❌ Wrong data types
reading_time: "10 minutes"      # Should be number
featured: "true"                # Should be boolean
tags: "web, tutorial"           # Should be list
date: 20240115                  # Should be string

# ✅ Correct data types
reading_time: 10
featured: true
tags: ["web", "tutorial"]
date: "2024-01-15"
---

Validation Checklist

  • YAML syntax is valid (no unescaped quotes, proper indentation)
  • Required fields are present (title, date for posts)
  • Date format is correct (YYYY-MM-DD)
  • Author ID exists in authors.yml (if using author field)
  • Category IDs exist in categories.yml (if using categories)
  • Tag IDs exist in tags.yml (if using tags)
  • Custom URL is unique (no duplicate custom_url values)
  • Image paths are correct (files exist in static directory)
  • Template exists (if using custom template)

Front Matter Extraction

# Extract and validate front matter
import yaml

def validate_front_matter(file_path):
    with open(file_path, 'r') as f:
        content = f.read()

    if content.startswith('---'):
        try:
            front_matter = content.split('---')[1]
            data = yaml.safe_load(front_matter)
            print("✅ Valid YAML")
            print(f"Fields: {list(data.keys())}")
            return data
        except yaml.YAMLError as e:
            print(f"❌ YAML Error: {e}")
    else:
        print("❌ No front matter found")

# Usage
validate_front_matter('content/posts/my-post.md')

Error Messages and Solutions

Error Cause Solution
"Invalid YAML syntax" Malformed YAML Check quotes, indentation, colons
"Missing required field: title" No title field Add title: "Your Title"
"Invalid date format" Wrong date format Use "YYYY-MM-DD" format
"Author ID not found" Invalid author reference Check authors.yml or remove author field
"Category ID not found" Invalid category ID Check categories.yml or remove categories
"Tag ID not found" Invalid tag ID Check tags.yml or remove tags
"Template not found" Missing custom template Create template file or remove template field
"Duplicate custom_url" URL conflict Use unique custom_url values

---
# ✅ Use clear, consistent naming

last_updated: "2024-01-20"   # Not lastUpdated or last_update

---

3. Required vs Optional Fields

---
# Always include these for posts
title: "Required Title"
date: "2024-01-15"

# Include these for better SEO

keywords: "relevant, keywords"

# Optional but helpful
author: 1
categories: [1]

featured_image: "images/hero.jpg"
---

Summary

Front matter is the foundation of content management in Stattic, providing:

Essential Functions

  • Content identification with titles and dates
  • Organization through categories, tags, and series
  • SEO optimization with metadata and descriptions
  • Display control with templates and featured flags
  • Custom functionality through extensible fields

Key Principles

  1. YAML syntax must be valid - use proper quotes and formatting
  2. Required fields are mandatory - title and date for posts
  3. Consistency is important - use same field names across content
  4. SEO matters - include descriptions and keywords
  5. Custom fields extend functionality - add fields as needed

Common Patterns

  • Blog posts: title, date, description, categories, tags, author
  • Static pages: title, description, custom_url, template, order
  • Tutorials: title, date, template, + custom fields like difficulty, reading_time
  • Landing pages: title, description, template, seo_title, featured_image

Best Practices

  • Use numeric IDs for authors, categories, and tags
  • Define all referenced entities in their respective YAML files
  • Include essential SEO fields (title, description, keywords)
  • Use consistent custom field naming
  • Test template integration for custom fields

Mastering front matter enables you to create well-organized, SEO-friendly, and highly functional static sites with Stattic.
