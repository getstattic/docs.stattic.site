Overview

Stattic uses Jinja2, a powerful Python templating engine, to generate HTML from your content and templates. This system provides:

  • Template inheritance for consistent layouts
  • Dynamic content rendering
  • Conditional logic and loops
  • Powerful filters for data transformation
  • Modular design with includes and macros

How Templates Work

Content (Markdown) + Template (Jinja2) = Generated HTML
  1. Content files (.md) provide data and text
  2. Templates (.html) define structure and layout
  3. Stattic combines them to generate static HTML files

Template Architecture

Template Hierarchy

Stattic uses a simple template system with the following templates:

templates/
├── base.html           # Foundation template
├── index.html          # Homepage template
├── post.html           # Individual post template
├── page.html           # Static page template
├── tag.html            # Tag listing template (basic)
├── category.html       # Category listing template (basic)
└── 404.html           # Error page template

Note: Stattic currently does not support complex archive systems, author pages, or partials/includes. Template organization is intentionally simple and focused.

Template Inheritance Flow

base.html (foundation)
├── index.html (homepage with pagination)
├── post.html (individual blog posts)
├── page.html (static pages)
├── tag.html (basic tag listings)
├── category.html (basic category listings)
└── 404.html (error page)

Template Selection Logic

Stattic automatically selects templates based on content type:

Content Type Template Used Fallback
Homepage index.html base.html
Blog post post.html base.html
Static page page.html base.html
Tag listing tag.html base.html
Category listing category.html base.html
404 page 404.html base.html

Note: Unlike complex CMSs, Stattic focuses on core functionality without advanced archive features.

Tag and Category Templates

Important: The current tag.html and category.html templates contain an error that may cause template rendering issues:

{# Current templates use (but this will cause errors): #}
<p>{{ post.excerpt | markdown }}</p>

{# Should be: #}
<p>{{ post.excerpt | safe }}</p>

This is because the markdown filter is not currently registered in Stattic's Jinja2 environment.

            </nav>

                {# Menu icon SVG #}

                    </ul>

The actual index.html template with pagination support:

{% extends "base.html" %}

{% block content %}
<main class="container max-w-2xl m-auto py-10 px-4" id="main-content">

    <ul class="space-y-6 stattic-content">
        {% for post in posts %}
        <li>
            <h2 class="text-4xl mb-2 font-bold font-heading-bold">
                <a href="{{ relative_path }}{{ post.permalink }}" class="text-links hover:underline">{{ post.title }}</a>

        </li>
        {% endfor %}
    </ul>

    {% if total_pages > 1 %}
    <nav class="mt-8">
      <ul class="flex gap-2 justify-center">
        {% if current_page > 1 %}
          <li>
            <a href="{% if current_page == 2 %}{{ relative_path }}{% else %}{{ relative_path }}page/{{ current_page - 1 }}/{% endif %}" 
               class="px-4 py-2 border rounded-md border-primary bg-transparent text-white hover:bg-slate-800 hover:text-white">
              Prev
            </a>
          </li>
        {% endif %}

        {% for link in page_numbers %}
          {% if link == '...' %}
            <li><span class="px-4 py-2 text-gray-400">…</span></li>
          {% elif link == current_page %}

              <span class="px-4 py-2 border border-primary rounded-md bg-slate-800 text-white">
                {{ link }}
              </span>
            </li>
          {% else %}
            <li>
              <a href="{% if link == 1 %}{{ relative_path }}{% else %}{{ relative_path }}page/{{ link }}/{% endif %}" 
                 class="px-4 py-2 border rounded-md border-primary bg-transparent text-white hover:bg-slate-800 hover:text-white">
                {{ link }}
              </a>
            </li>
          {% endif %}
        {% endfor %}

        {% if current_page < total_pages %}
          <li>
            <a href="{{ relative_path }}page/{{ current_page + 1 }}/" 
               class="px-4 py-2 border rounded-md border-primary bg-transparent text-white hover:bg-slate-800 hover:text-white">
              Next
            </a>
          </li>
        {% endif %}
      </ul>
    </nav>
    {% endif %}

</main>
{% endblock %}

Creating Custom Templates

Custom Templates via Front Matter

Stattic supports specifying custom templates directly in your content's front matter. This allows you to create specialized layouts for specific posts or pages.

How It Works

Add a template field to your front matter:

---
title: "My Special Post"
template: "featured"
---

Content goes here...

Stattic will automatically look for:

  • Posts: post-{template}.html (e.g., post-featured.html)
  • Pages: page-{template}.html (e.g., page-featured.html)

If the custom template doesn't exist, it falls back to the default post.html or page.html.

Example Custom Post Template

Create templates/post-featured.html:

{% extends "base.html" %}

{% block content %}


        {# Featured badge #}

            ⭐ FEATURED


        {% if page.featured_image %}

        {% endif %}

        {{ title }}

            {{ date }} · 
            {{ get_author_name(author) }}


        {{ content | safe }}

        {# Call-to-action for featured posts #}

            Enjoyed this featured post?
            Check out more of our featured content.
            [                Browse All Posts            ]({{ relative_path }}blog/)



{% endblock %}

Example Custom Page Template

Create templates/page-landing.html:

{% extends "base.html" %}

{% block content %}

    {# Hero section #}

        {{ title }}
        {% if page.description %}
        {{ page.description }}
        {% endif %}
        [            Get Started        ](#content)


    {# Content section #}

        {{ content | safe }}


{% endblock %}

Template Naming Convention

Content Type Front Matter Template File
Blog Post template: "featured" post-featured.html
Blog Post template: "tutorial" post-tutorial.html
Static Page template: "landing" page-landing.html
Static Page template: "about" page-about.html

Basic Loops and Conditionals

Looping Through Posts

{# Loop through posts on index page #}

<li>
    <h2 class="text-4xl mb-2 font-bold font-heading-bold">

{% if posts %}
    {% for post in posts %}
    <h3>{{ post.title }}</h3>
    {% endfor %}
{% else %}
    <p>No posts found.</p>
{% endif %}

        {% endfor %}

Troubleshooting

Common Template Issues

1. Template Not Found

Error: Template 'custom.html' not found

Solutions:

  • Check file exists in templates/ directory
  • Verify file name spelling and case
  • Ensure you're using correct template naming (post-custom.html for posts)

2. Variable Not Defined

Error: 'post' is undefined

Solutions:

{# ✅ Check variable exists #}
{% if title %}
<h1>{{ title }}</h1>
{% endif %}

{# ✅ Use default values #}
{{ title | default('Untitled') }}

{# ✅ Remember Stattic's variable names #}
{{ title }}           
{{ content | safe }}

3. Markdown Filter Error

Error: No filter named 'markdown'

Solutions:

{# ❌ This will cause errors #}
{{ post.excerpt | markdown }}

{# ✅ Use safe filter for HTML content #}
{{ post.excerpt | safe }}

Note: The markdown filter is not currently registered in Stattic's template environment.

4. Page Variable Undefined Error

Error: 'page' is undefined

Common Cause: Using {{ page.permalink }} in base template without checking if page exists.

Solutions:

{# ❌ This will cause errors in index/listing templates #}
<meta property="og:url" content="{{ site_url }}/{{ page.permalink }}">

{# ✅ Always check if page variable exists first #}
<meta property="og:url" content="{% if canonical %}{{ canonical }}{% elif page and page.permalink %}{{ site_url }}/{{ page.permalink | trim('/') }}{% elif site_url %}{{ site_url }}{% endif %}">

Note: The page variable is only available in individual post/page templates, not in index or listing templates.

5. Content Not Displaying

{# ❌ Missing | safe filter #}
{{ content }}

{# ✅ Always use | safe for HTML content #}
{{ content | safe }}

6. Relative Path Issues

{# ❌ Hard-coded paths #}
<link href="/assets/css/style.css" rel="stylesheet">

{# ✅ Use relative_path variable #}
<link href="{{ relative_path }}assets/css/style.css" rel="stylesheet">

Summary

Stattic's template system provides essential tools for creating clean, functional websites:

Key Features

  • Jinja2 templating with inheritance support
  • Simple variable context with core site data
  • Custom templates via front matter
  • Built-in pagination for index pages
  • Clean asset paths with relative_path support
  • Dynamic navigation based on page metadata

Template Types

  • Base template (base.html) for site-wide layout
  • Content templates (post.html, page.html) for individual content
  • Index template (index.html) for homepage with pagination
  • Archive templates (tag.html, category.html) for listings
  • Custom templates (post-custom.html, page-custom.html) for specialized layouts
  • Error template (404.html) for error pages

Best Practices

  1. Keep templates simple and focused
  2. Use semantic HTML for accessibility
  3. Handle missing data gracefully
  4. Use relative paths correctly
  5. Debug systematically with comments
  6. Use | safe for HTML content instead of missing | markdown filter

Variable Access

  • Access content via {{ title }}, {{ content | safe }}, {{ author }}
  • Use {{ relative_path }} for asset links
  • Access metadata via {{ page.field_name }}
  • Resolve authors with {{ get_author_name(author) }}
  • Navigate pages with {% for page in pages %} (filtered by nav_hide)

Known Issues

  • Markdown filter not registered: Use {{ content | safe }} instead of {{ content | markdown }}
  • Tag/category templates may have errors due to missing markdown filter
  • No strftime filter available for date formatting
  • Page variable undefined errors: Always check {% if page and page.permalink %} before accessing page properties in base templates

The template system scales from simple blogs to custom page layouts while maintaining clean, maintainable code that focuses on core functionality rather than complex features.
