Overview

Stattic uses Jinja2, a powerful Python templating engine, to generate HTML from your content and templates. This system provides:

  • Template inheritance for consistent layouts
  • Dynamic content rendering
  • Conditional logic and loops
  • Powerful filters for data transformation
  • Modular design with includes and macros

How Templates Work

Content (Markdown) + Template (Jinja2) = Generated HTML
  1. Content files (.md) provide data and text
  2. Templates (.html) define structure and layout
  3. Stattic combines them to generate static HTML files

Template Architecture

Template Hierarchy

Stattic uses a simple template system with the following templates:

templates/
├── base.html           # Foundation template
├── index.html          # Homepage template
├── post.html           # Individual post template
├── page.html           # Static page template
├── tag.html            # Tag listing template (basic)
├── category.html       # Category listing template (basic)
└── 404.html           # Error page template

Note: Stattic currently does not support complex archive systems, author pages, or partials/includes. Template organization is intentionally simple and focused.

Template Inheritance Flow

base.html (foundation)
├── index.html (homepage with pagination)
├── post.html (individual blog posts)
├── page.html (static pages)
├── tag.html (basic tag listings)
├── category.html (basic category listings)
└── 404.html (error page)

Template Selection Logic

Stattic automatically selects templates based on content type:

Content Type Template Used Fallback
Homepage index.html base.html
Blog post post.html base.html
Static page page.html base.html
Tag listing tag.html base.html
Category listing category.html base.html
404 page 404.html base.html

Note: Unlike complex CMSs, Stattic focuses on core functionality without advanced archive features.

Default Templates

Base Template Structure

The base.html template provides the foundation. Here's the actual structure used by Stattic:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>{{ ((seo_title or title) ~ ' | ' ~ (site_title or 'Stattic')) if (seo_title or title) else (site_title or 'Stattic') }}</title>
    <meta name="description" content="{% if seo_description %}{{ seo_description }}{% else %}Stattic is a fast, lightweight static site generator.{% endif %}">
    <meta name="keywords" content="{% if seo_keywords %}{{ seo_keywords }}{% else %}SSG, Static Site Generator{% endif %}">
    <meta name="author" content="{% if author %}{{ author }}{% else %}Robert DeVore{% endif %}">

    {# CSS Assets #}
    <link href="{{ relative_path }}assets/css/tailwind.min.css" rel="stylesheet">
    {% if minify %}
    <link href="{{ relative_path }}assets/css/stattic.min.css" rel="stylesheet">
    {% else %}
    <link href="{{ relative_path }}assets/css/stattic.css" rel="stylesheet">
    <link href="{{ relative_path }}assets/css/fonts.css" rel="stylesheet">
    {% endif %}

    {# Open Graph Tags #}
    <meta property="og:title" content="{% if seo_title %}{{ seo_title }}{% else %}{{ title }}{% endif %}">
    <meta property="og:description" content="{% if seo_description %}{{ seo_description }}{% else %}Stattic is a fast, lightweight static site generator.{% endif %}">
    <meta property="og:url" content="{% if canonical %}{{ canonical }}{% elif page and page.permalink %}{{ site_url }}/{{ page.permalink | trim('/') }}{% elif site_url %}{{ site_url }}{% endif %}">
    {% if featured_image %}
    <meta property="og:image" content="{{ relative_path }}{{ featured_image }}">
    {% endif %}

    {# JavaScript #}
    <script src="{{ relative_path }}assets/js/alpine.min.js" defer></script>
</head>
<body class="bg-background bg-[radial-gradient(ellipse_at_center,_var(--tw-gradient-stops))] from-slate-900 to-slate-950 text-primary font-body-light">
    {# Skip Links for Accessibility #}
    <a href="#main-content" class="skip-link">Skip to main content</a>
    <a href="#navigation" class="skip-link">Skip to navigation</a>

    {# Header with Mobile Menu #}
    <header id="navigation" x-data="{ mobileMenuOpen: false }">
        <div class="container max-w-screen-xl m-auto py-6 px-4 flex justify-between items-center">
            <h2 class="text-3xl font-bold font-heading-bold">
                <a href="{{ relative_path }}" class="text-links">{{ site_title or 'Stattic' }}</a>
            </h2>

            {# Desktop Navigation #}
            <nav class="hidden md:flex space-x-6">
                <a href="{{ relative_path }}" class="text-links hover:underline">Home</a>
                <a href="{{ relative_path }}blog/" class="text-links hover:underline">Blog</a>
            </nav>

            {# Mobile Menu Button #}
            <button @click="mobileMenuOpen = !mobileMenuOpen" class="md:hidden text-links">
                <svg class="w-6 h-6" fill="none" stroke="currentColor" viewBox="0 0 24 24">
                    <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M4 6h16M4 12h16M4 18h16"></path>
                </svg>
            </button>
        </div>

        {# Mobile Menu #}
        <div x-show="mobileMenuOpen" x-transition class="md:hidden bg-slate-800 border-t border-slate-700">
            <nav class="container max-w-screen-xl m-auto px-4 py-4 space-y-2">
                <a href="{{ relative_path }}" class="block text-links hover:underline">Home</a>
                <a href="{{ relative_path }}blog/" class="block text-links hover:underline">Blog</a>
            </nav>
        </div>
    </header>

    {# Main content area #}
    {% block content %}
    <!-- Child templates override this block -->
    {% endblock %}

    {# Footer #}
    <footer class="bg-slate-900 text-center py-8 border-t border-slate-700">
        <div class="container max-w-screen-xl m-auto px-4">
            <p class="text-slate-400">&copy; {{ "now" | strftime("%Y") }} {{ site_title or 'Stattic' }}. All rights reserved.</p>
        </div>
    </footer>
</body>
</html>

Index Template

The actual index.html template with pagination support:

{% extends "base.html" %}

{% block content %}
<main class="container max-w-2xl m-auto py-10 px-4" id="main-content">
    <ul class="space-y-6 stattic-content">
        {% for post in posts %}
        <li>
            <h2 class="text-4xl mb-2 font-bold font-heading-bold">
                <a href="{{ relative_path }}{{ post.permalink }}" class="text-links hover:underline">{{ post.title }}</a>
            </h2>
            <div class="dark:text-slate-50 mb-2">{{ post.excerpt | safe }}</div>
        </li>
        {% endfor %}
    </ul>

    {# Pagination - only shown if multiple pages #}
    {% if total_pages > 1 %}
    <nav class="mt-8">
      <ul class="flex gap-2 justify-center">
        {% if current_page > 1 %}
          <li>
            <a href="{% if current_page == 2 %}{{ relative_path }}{% else %}{{ relative_path }}page/{{ current_page - 1 }}/{% endif %}" 
               class="px-4 py-2 border rounded-md border-primary bg-transparent text-white hover:bg-slate-800 hover:text-white">
              Prev
            </a>
          </li>
        {% endif %}

        {% for link in page_numbers %}
          {% if link == '...' %}
            <li><span class="px-4 py-2 text-gray-400">…</span></li>
          {% elif link == current_page %}
            <li>
              <span class="px-4 py-2 border border-primary rounded-md bg-slate-800 text-white">
                {{ link }}
              </span>
            </li>
          {% else %}
            <li>
              <a href="{% if link == 1 %}{{ relative_path }}{% else %}{{ relative_path }}page/{{ link }}/{% endif %}" 
                 class="px-4 py-2 border rounded-md border-primary bg-transparent text-white hover:bg-slate-800 hover:text-white">
                {{ link }}
              </a>
            </li>
          {% endif %}
        {% endfor %}

        {% if current_page < total_pages %}
          <li>
            <a href="{{ relative_path }}page/{{ current_page + 1 }}/" 
               class="px-4 py-2 border rounded-md border-primary bg-transparent text-white hover:bg-slate-800 hover:text-white">
              Next
            </a>
          </li>
        {% endif %}
      </ul>
    </nav>
    {% endif %}
</main>
{% endblock %}

{# Page/Post data #}

Post/Page Variables

Available in individual post and page templates:

{# Content basics #}
{{ title }}                <!-- Post/page title -->
{{ content }}              <!-- HTML content (always use | safe) -->
{{ date }}                 <!-- Formatted date for display -->
{{ author }}               <!-- Author name or get_author_name(author) -->

{# Metadata (from front matter) #}
{{ metadata }}             <!-- Full metadata object -->
{{ page }}                 <!-- Alias for metadata -->
{{ page.featured_image }}  <!-- Featured image path -->
{{ featured_image }}       <!-- Direct featured image access -->

{# SEO fields #}
{{ seo_title }}            <!-- SEO title override -->
{{ seo_description }}      <!-- SEO description -->
{{ seo_keywords }}         <!-- SEO keywords -->

{# Organization #}
{{ categories }}           <!-- List of category names -->
{{ tags }}                 <!-- List of tag names -->

{# Template utilities #}
{{ relative_path }}        <!-- Path to root directory -->
{{ get_author_name }}      <!-- Function to resolve author names -->

Index Template Variables

Available in the homepage template:

{# Post listings #}
{{ posts }}                <!-- Current page's posts -->

{# Pagination #}
{{ current_page }}         <!-- Current page number (1, 2, 3...) -->
{{ total_pages }}          <!-- Total number of pages -->
{{ page_numbers }}         <!-- Array of page links -->

{# Navigation #}
{{ relative_path }}        <!-- Relative path for links -->

{# Site data #}
{{ site_title }}           <!-- Site title -->
{{ site_tagline }}         <!-- Site tagline -->

Important Notes:

  • Stattic does not have complex post objects with .author.name style access
  • Author information is accessed via get_author_name(author) function
  • Tags and categories are simple arrays of names, not objects with .slug or .id
  • No built-in reading time, related posts, or advanced metadata features

Creating Custom Templates

Custom Templates via Front Matter

Stattic supports specifying custom templates directly in your content's front matter. This allows you to create specialized layouts for specific posts or pages.

How It Works

Add a template field to your front matter:

---
title: "My Special Post"
template: "featured"
---

Content goes here...

Stattic will automatically look for:

  • Posts: post-{template}.html (e.g., post-featured.html)
  • Pages: page-{template}.html (e.g., page-featured.html)

If the custom template doesn't exist, it falls back to the default post.html or page.html.

Example Custom Post Template

Create templates/post-featured.html:

{% extends "base.html" %}

{% block content %}
<main class="container max-w-4xl m-auto py-10 px-4" id="main-content">
    <article class="prose dark:prose-dark max-w-none stattic-content">
        {# Featured badge #}
        <div class="bg-yellow-500 text-black px-3 py-1 rounded-full text-sm font-bold mb-4 inline-block">
            ⭐ FEATURED
        </div>

        {% if page.featured_image %}
        <img src="/{{ page.featured_image }}" alt="Featured Image for {{ page.title }}" class="featured-image" />
        {% endif %}

        <h1 class="text-6xl font-bold mb-4 text-links font-heading-bold post-title">{{ title }}</h1>
        <p class="text-2md post-meta mb-4 pb-2 text-accent italic border-b border-primary">
            <span class="date">{{ date }}</span> &middot; 
            <span class="author">{{ get_author_name(author) }}</span>
        </p>

        {{ content | safe }}

        {# Call-to-action for featured posts #}
        <div class="bg-slate-800 p-8 rounded-lg text-center mt-12">
            <h3 class="text-2xl font-bold mb-4">Enjoyed this featured post?</h3>
            <p class="text-slate-300 mb-6">Check out more of our featured content.</p>
            <a href="{{ relative_path }}blog/" class="bg-blue-600 text-white px-6 py-3 rounded-lg hover:bg-blue-700">
                Browse All Posts
            </a>
        </div>
    </article>
</main>
{% endblock %}

Example Custom Page Template

Create templates/page-landing.html:

{% extends "base.html" %}

{% block content %}
<main class="container max-w-6xl m-auto py-10 px-4" id="main-content">
    {# Hero section #}
    <section class="text-center py-16 mb-12">
        <h1 class="text-6xl font-bold mb-4 text-links font-heading-bold">{{ title }}</h1>
        {% if page.description %}
        <p class="text-2xl text-slate-300 mb-8">{{ page.description }}</p>
        {% endif %}
        <a href="#content" class="bg-blue-600 text-white px-8 py-4 rounded-lg font-bold hover:bg-blue-700">
            Get Started
        </a>
    </section>

    {# Content section #}
    <section id="content" class="prose dark:prose-dark max-w-none stattic-content">
        {{ content | safe }}
    </section>
</main>
{% endblock %}

Content Type Front Matter Template File

Template Inheritance

Basic Inheritance

Stattic templates use Jinja2's inheritance system. All templates extend from base.html:

{# Child template extends parent #}
{% extends "base.html" %}

{# Override the content block #}
{% block content %}
<h1>Custom Content</h1>
{% endblock %}

Available Blocks

The base.html template provides one main block:

{% block content %}
<!-- Child templates override this block -->
{% endblock %}

Simple Template Inheritance

{# templates/post.html #}
{% extends "base.html" %}

{% block content %}
<main class="container max-w-2xl m-auto py-10 px-4" id="main-content">
    <article class="prose dark:prose-dark max-w-none stattic-content">
        <h1>{{ title }}</h1>
        {{ content | safe }}
    </article>
</main>
{% endblock %}

Note: Stattic uses a simple inheritance model focused on the main content area. Advanced block overrides and complex inheritance patterns are not commonly used.

Basic Loops and Conditionals

Looping Through Posts

{# Loop through posts on index page #}
{% for post in posts %}
<article>
    <h2><a href="{{ relative_path }}{{ post.permalink }}">{{ post.title }}</a></h2>
    <div>{{ post.excerpt | safe }}</div>
</article>
{% endfor %}

{# Handle empty post lists #}
{% if posts %}
    {% for post in posts %}
    <article>{{ post.title }}</article>
    {% endfor %}
{% else %}
    <p>No posts found.</p>
{% endif %}

Conditional Logic

{# Check for featured image #}
{% if page.featured_image %}
<img src="/{{ page.featured_image }}" alt="Featured Image for {{ page.title }}" />
{% endif %}

{# Check for metadata fields #}
{% if seo_description %}
<meta name="description" content="{{ seo_description }}">
{% else %}
<meta name="description" content="Default description">
{% endif %}

{# Show pagination only if multiple pages #}
{% if total_pages > 1 %}
<nav class="pagination">
    <!-- pagination links -->
</nav>
{% endif %}

Pagination Logic

{# Pagination controls #}
{% if current_page > 1 %}
<a href="{% if current_page == 2 %}{{ relative_path }}{% else %}{{ relative_path }}page/{{ current_page - 1 }}/{% endif %}">
    Previous
</a>
{% endif %}

{% if current_page < total_pages %}
<a href="{{ relative_path }}page/{{ current_page + 1 }}/">Next</a>
{% endif %}

Basic Filters

Common Jinja2 Filters

Stattic supports standard Jinja2 filters:

{# Content filters #}
{{ content | safe }}              <!-- Don't escape HTML -->
{{ title | upper }}               <!-- UPPERCASE -->
{{ title | lower }}               <!-- lowercase -->
{{ title | title }}               <!-- Title Case -->

{# Default values #}
{{ author | default('Anonymous') }}
{{ site_title | default('My Site') }}

{# String manipulation #}
{{ content | length }}            <!-- Character count -->
{{ title | truncate(50) }}        <!-- Truncate to 50 chars -->

{# List operations #}
{{ categories | length }}         <!-- Number of categories -->
{{ tags | join(', ') }}          <!-- Comma-separated list -->

Custom Stattic Functions

{# Author name resolution #}
{{ get_author_name(author) }}     <!-- Resolve author ID to name -->

{# Date formatting (built-in to Stattic) #}
{{ date }}                        <!-- Pre-formatted date string -->

Note: Stattic does not include complex custom filters or advanced template functions. Keep templating simple and rely on standard Jinja2 functionality.

Best Practices

Template Organization

1. Keep Templates Simple

templates/
├── base.html              # Foundation template
├── index.html             # Homepage
├── post.html              # Blog posts
├── page.html              # Static pages
├── post-featured.html     # Custom post template
├── page-landing.html      # Custom page template
├── tag.html               # Tag listings
├── category.html          # Category listings
└── 404.html              # Error page

2. Use Descriptive Names

  • post-featured.html instead of special.html
  • page-landing.html instead of custom.html
  • Clear template names make maintenance easier

Code Quality

1. Use Semantic HTML

{# ✅ Good semantic structure #}
<article class="post">
    <header class="post-header">
        <h1 class="post-title">{{ title }}</h1>
        <time class="post-date" datetime="{{ date }}">{{ date }}</time>
    </header>

    <div class="post-content">
        {{ content | safe }}
    </div>
</article>

{# ❌ Avoid generic divs #}
<div class="post">
    <div class="title">{{ title }}</div>
    <div class="date">{{ date }}</div>
    <div class="content">{{ content | safe }}</div>
</div>

2. Handle Missing Data

{# ✅ Always check for data existence #}
{% if page.featured_image %}
<img src="/{{ page.featured_image }}" alt="Featured Image for {{ page.title }}" />
{% endif %}

{# ✅ Provide fallbacks #}
<meta name="author" content="{% if author %}{{ author }}{% else %}{{ site_title }}{% endif %}">

3. Optimize Performance

{# ✅ Use relative paths correctly #}
<link href="{{ relative_path }}assets/css/tailwind.min.css" rel="stylesheet">

{# ✅ Efficient conditionals #}
{% if posts %}
    {% for post in posts %}
    <article>{{ post.title }}</article>
    {% endfor %}
{% else %}
    <p>No posts found.</p>
{% endif %}

Accessibility

1. Proper Headings

{# ✅ Logical heading hierarchy #}
<h1>{{ title }}</h1>              <!-- Page/post title -->
<h2>Section Title</h2>             <!-- Major sections -->
<h3>Subsection</h3>                <!-- Subsections -->

{# ❌ Avoid skipping levels #}
<h1>Page Title</h1>
<h3>Subsection</h3>  <!-- Missing h2 -->

2. Alt Text and ARIA

{# ✅ Descriptive alt text #}
<img src="/{{ page.featured_image }}" alt="Featured Image for {{ page.title }}" />

{# ✅ Skip links for navigation #}
<a href="#main-content" class="skip-link">Skip to main content</a>

{# ✅ Proper navigation labels #}
<nav aria-label="Main navigation">
    <a href="{{ relative_path }}">Home</a>
    <a href="{{ relative_path }}blog/">Blog</a>
</nav>

3. Focus Management

{# ✅ Proper focus indicators in CSS #}
<style>
a:focus, button:focus {
    outline: 2px solid var(--primary-color);
    outline-offset: 2px;
}
</style>

Troubleshooting

Common Template Issues

1. Template Not Found

Error: Template 'custom.html' not found

Solutions:

  • Check file exists in templates/ directory
  • Verify file name spelling and case
  • Ensure you're using correct template naming (post-custom.html for posts)

2. Variable Not Defined

Error: 'post' is undefined

Solutions:

{# ✅ Check variable exists #}
{% if title %}
{{ title }}
{% endif %}

{# ✅ Use default values #}
{{ title | default('Untitled') }}

{# ✅ Remember Stattic's variable names #}
{{ title }}           <!-- Not {{ post.title }} -->
{{ content | safe }}  <!-- Not {{ post.content }} -->

3. Content Not Displaying

{# ❌ Missing | safe filter #}
{{ content }}

{# ✅ Always use | safe for HTML content #}
{{ content | safe }}

4. Relative Path Issues

{# ❌ Hard-coded paths #}
<link href="/assets/css/style.css" rel="stylesheet">

{# ✅ Use relative_path variable #}
<link href="{{ relative_path }}assets/css/style.css" rel="stylesheet">

Summary

Stattic's template system provides essential tools for creating clean, functional websites:

Key Features

  • Jinja2 templating with inheritance support
  • Simple variable context with core site data
  • Custom templates via front matter
  • Built-in pagination for index pages
  • Clean asset paths with relative_path support

Template Types

  • Base template (base.html) for site-wide layout
  • Content templates (post.html, page.html) for individual content
  • Index template (index.html) for homepage with pagination
  • Custom templates (post-custom.html, page-custom.html) for specialized layouts
  • Error template (404.html) for error pages

Best Practices

  1. Keep templates simple and focused
  2. Use semantic HTML for accessibility
  3. Handle missing data gracefully
  4. Use relative paths correctly
  5. Debug systematically with comments

Variable Access

  • Access content via {{ title }}, {{ content | safe }}, {{ author }}
  • Use {{ relative_path }} for asset links
  • Access metadata via {{ page.field_name }}
  • Resolve authors with {{ get_author_name(author) }}

The template system scales from simple blogs to custom page layouts while maintaining clean, maintainable code that focuses on core functionality rather than complex features.
