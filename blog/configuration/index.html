Overview

Stattic can be configured in two ways:

  1. Configuration File (stattic.yml) - Recommended for project settings
  2. Command Line Options - Great for overrides and automation

Configuration files provide persistent settings, while command-line options allow for temporary overrides and automation scripts.

Configuration Methods

Method 1: Configuration File (Recommended)

Create a stattic.yml file in your project root:

stattic --init yml

This creates a stattic.yml file with all available settings and sets up your complete project structure.

Supported file formats:

  • stattic.yml (YAML - recommended)
  • stattic.yaml (YAML alternative)
  • stattic.json (JSON format)

Generated stattic.yml example:

# Stattic Configuration File
# Configure your static site generator settings here

# Site information
site_url: https://example.com
site_title: My Static Site
site_tagline: Built with Stattic

# Build settings
output: output
content: content
templates: templates
assets: assets
blog_slug: blog

# Content settings
posts_per_page: 5
sort_by: date  # date, title, author, order

# Styling
fonts:
  - Quicksand
  - Open Sans

# SEO settings
robots: public  # public or private

# Development settings
minify: false
watch: false

Note: The generated config file doesn't include all available settings. You can manually add llms: public or any other settings from the complete reference below.

Method 2: Command Line Options

Override any setting via command line:

# Build with custom settings
stattic --site-title "My Blog" --posts-per-page 10 --minify

# Use different directories
stattic --content src --output dist --assets static

# Override site URL for production
stattic --site-url https://example.com

Method 3: Combined Approach

Use config file for base settings, command line for overrides:

# Config file has minify: false, but enable for production
stattic --minify

# Override site URL for staging
stattic --site-url https://staging.example.com

Priority Order (highest to lowest):

  1. Command line arguments
  2. Configuration file settings
  3. Default values

  - Open Sans

# SEO settings

Command Line Options

All Available Options

stattic [OPTIONS]

OPTIONS:
  --output DIR              Output directory for generated site
  --content DIR             Content directory containing markdown files  
  --templates DIR           Templates directory
  --assets DIR              Assets directory to copy to output
  --posts-per-page NUM      Number of posts per page for pagination
  --sort-by FIELD           Sort posts by: date, title, author, order
  --fonts FONTS             Comma-separated list of Google Fonts
  --site-title TITLE        Site title for metadata
  --site-tagline TAGLINE    Site tagline for metadata
  --site-url URL            Site URL for RSS feeds and sitemaps
  --robots MODE             Robots.txt configuration: public or private
  --llms MODE               LLMs.txt configuration: public or private
  --minify                  Minify CSS and JS assets
  --blog-slug SLUG          Custom slug for posts instead of 'blog'
  --init FORMAT             Create sample config file: yml, yaml, or json
  --version                 Show version number
  --help                    Show help message

Basic Usage

# Build site with default settings
stattic

# Build with custom output directory
stattic --output dist

# Build with different content directory
stattic --content src/posts

Initialization Command

Create a sample configuration file and complete project structure:

# Create YAML config (recommended)
stattic --init yml

# Create YAML config (alternative)
stattic --init yaml

# Create JSON config
stattic --init json

The --init command creates:

  • Configuration file (stattic.yml, stattic.yaml, or stattic.json)
  • Complete directory structure (templates/, content/, assets/)
  • All default templates (base.html, post.html, index.html, etc.)
  • Sample content files (sample-post.md, sample-page.md)
  • Configuration files (authors.yml, categories.yml, tags.yml)

Note: The generated stattic.yml file includes most configuration options but may not include all available settings (such as llms). You can add any missing settings manually using the reference below.

Build Configuration

Directory Structure

# Default structure
output: output           # Generated site goes here
content: content         # Your .md files
templates: templates     # Custom templates (falls back to built-in if missing)
assets: assets          # CSS, JS, images to copy (falls back to 'assets' if exists)

# Custom structure
output: dist            # Different output directory
content: src/posts      # Posts in src/posts/
assets: static          # Assets in static/

Assets Directory Behavior:

  • If assets is not specified (None), Stattic looks for an 'assets' directory
  • If the specified assets directory doesn't exist, no error occurs
  • Templates directory falls back to built-in templates if custom directory is missing

Content Settings

Posts Per Page

Controls pagination on homepage and archive pages:

posts_per_page: 5       # Default: 5 posts per page
posts_per_page: 10      # More posts per page
posts_per_page: 1       # One post per page

Recommended values:

  • Blogs: 5-10 posts per page
  • News sites: 15-20 posts per page
  • Photo blogs: 5-8 posts per page

Post Sorting

Control how posts are ordered:

sort_by: date           # Latest first (default)
sort_by: title          # Alphabetical by title
sort_by: author         # Grouped by author
sort_by: order          # Manual order (use 'order' in front matter)

Using Manual Order:

---
title: "Important Post"
order: 1                # Will appear first
---

---
title: "Regular Post"
order: 10               # Will appear later
---

Blog URL Structure

Customize the URL structure for your posts:

blog_slug: blog         # Posts at /blog/post-name/ (default)
blog_slug: posts        # Posts at /posts/post-name/
blog_slug: articles     # Posts at /articles/post-name/
blog_slug: news         # Posts at /news/post-name/

SEO and Crawlers

Robots.txt Configuration

Control search engine crawling:

robots: public          # Allow all crawlers (default)
robots: private         # Block all crawlers

Generated robots.txt (public):

User-agent: *
Allow: /
Sitemap: https://example.com/sitemap.xml

Generated robots.txt (private):

User-agent: *
Disallow: /

LLMs.txt for AI Crawlers

Control AI/LLM crawler access:

llms: public           # Allow AI crawlers (default)
llms: private          # Block AI crawlers

Note: This setting is not included in the generated config file from stattic --init yml, but you can add it manually. The default value is public if not specified.

This generates an llms.txt file that provides structured information about your site for AI crawlers.

Development Settings

Minification

Control asset optimization:

minify: true           # Minify CSS and JS (production)
minify: false          # Keep assets readable (development, default)

What gets minified:

  • assets/css/*.css → assets/css/*.min.css
  • assets/js/*.js → assets/js/*.min.js

Command line override:

# Force minification regardless of config
stattic --minify

Watch Mode

watch: false           # Default (not implemented yet)

Note: Watch mode is defined in the configuration but not yet implemented in the current version. Currently, stattic runs a single build process.

Environment-Specific Configs

Development vs Production

Since Stattic only looks for stattic.yml, stattic.yaml, or stattic.json, you can manage environments through:

  1. Multiple config files with manual switching
  2. Command line overrides
  3. Environment variables (if supported by your deployment)

Development Workflow

stattic.dev.yml (manual switching):

# Development settings
site_url: http://localhost:8000
site_title: My Site (DEV)
site_tagline: Local development version

# Development optimizations
minify: false           # Easier debugging
robots: private         # Don't index development

# Local directories
output: output
content: content
assets: assets

Copy for development:

cp stattic.dev.yml stattic.yml
stattic

Production Build

Command line overrides for production:

# Override for production without changing config file
stattic --site-url https://example.com --minify --robots public

CI/CD Integration

GitHub Actions example:

# .github/workflows/build.yml
- name: Build for staging
  run: stattic --site-url https://staging.example.com --robots private

- name: Build for production  
  run: stattic --site-url https://example.com --minify --robots public

Advanced Configuration

Google Fonts Configuration

Stattic automatically downloads and optimizes Google Fonts:

fonts:
  - Inter               # Single font family
  - Roboto
  - Source Code Pro

# Or empty for no Google Fonts
fonts: []

What Stattic does with fonts:

  1. Downloads font files from Google Fonts
  2. Creates local fonts.css file
  3. Updates templates to reference local fonts
  4. Improves performance by eliminating external requests

Font Selection Tips:

  • Body text: Inter, Roboto, Open Sans, Lato
  • Headings: Poppins, Montserrat, Playfair Display
  • Code: Source Code Pro, JetBrains Mono, Fira Code
  • Limit to 2-3 fonts for performance

Custom Directory Structures

Monorepo Setup:

# Site in subdirectory of larger project
content: website/content
templates: website/templates
assets: website/static
output: website/build

Separate Content Repository:

# Content in different location
content: ../content-repo/posts
assets: ../content-repo/assets
output: ../deploy/public

Build Script Integration

package.json scripts:

### Configuration Examples for Different Needs

**Development script (`package.json`):**
```json
{
  "scripts": {
    "build": "stattic",
    "build:prod": "stattic --minify",
    "build:dev": "stattic --site-url http://localhost:8000 --robots private"
  }
}

Shell script for different environments:

#!/bin/bash
# build.sh

case "$1" in
  dev)
    stattic --robots private --minify false
    ;;
  staging)
    stattic --site-url https://staging.example.com --robots private --minify
    ;;
  prod)
    stattic --site-url https://example.com --robots public --minify
    ;;
  *)
    echo "Usage: $0 {dev|staging|prod}"
    exit 1
esac

Troubleshooting

Common Configuration Issues

1. Site URL Problems

Problem: RSS feeds or sitemaps not generating

# ❌ Missing or invalid site_url
site_url: ""
# or
site_url: localhost:8000

Solution: Provide complete URL with protocol

# ✅ Complete URL with protocol
site_url: https://example.com

2. Directory Path Issues

Problem: Build fails with "directory not found"

# ❌ Directory doesn't exist
content: posts
assets: static

Solution: Verify directories exist or use --init to create structure

# Create complete project structure
stattic --init yml

3. Configuration File Not Found

Problem: Stattic doesn't load your config

  • Check filename: must be stattic.yml, stattic.yaml, or stattic.json
  • Check location: must be in current working directory
  • Check syntax: run YAML validation

Solution:

# Verify file exists
ls -la stattic.yml

# Check YAML syntax
python3 -c "import yaml; yaml.safe_load(open('stattic.yml'))"

4. Font Loading Issues

Problem: Fonts not downloading or loading

# ❌ Invalid font names
fonts:
  - "Inter Font"
  - "Source-Code-Pro"

Solution: Use exact Google Fonts family names

# ✅ Correct font names
fonts:
  - Inter
  - Source Code Pro
  - JetBrains Mono

5. Minification Problems

Problem: Site breaks with minification enabled

Solution: Check for syntax errors in CSS/JS

# Build without minification to debug (remove --minify flag)
stattic

# Check specific asset files for syntax errors

Validation Commands

Check configuration file:

# Test YAML syntax
python3 -c "import yaml; print('✅ Valid YAML')" && \
python3 -c "import yaml; yaml.safe_load(open('stattic.yml'))" 2>/dev/null && \
echo "✅ Configuration file is valid" || echo "❌ Configuration file has errors"

# Test build with current config
stattic --help  # Verify stattic is working
stattic         # Test actual build

Debug build process:

# Check what settings are being used
stattic --help
stattic --version

Common Error Messages

Error Cause Solution
"Content directory not found" Missing content folder Create content/ directory
"No posts found" Empty content directory Add .md files to content
"Invalid YAML syntax" Malformed config file Check YAML syntax
"Font download failed" Network/font name issue Check font names and connection
"Minification failed" CSS/JS syntax error Fix syntax or disable minification

Configuration Best Practices

1. Start Simple

Begin with minimal configuration:

site_title: My Site
site_url: https://example.com

Add settings as needed.

2. Use Environment-Specific Configs

Don't mix development and production settings:

  • stattic.dev.yml for development
  • stattic.prod.yml for production
  • stattic.yml as default

3. Document Your Settings

Add comments to explain custom settings:

# Custom blog slug for better SEO
blog_slug: articles

# Higher posts per page for fast loading
posts_per_page: 6

# Fonts chosen for readability
fonts:
  - Inter      # Body text
  - Fira Code  # Code blocks

4. Version Control

Include configuration in version control:

# Include configs
stattic.yml
stattic.*.yml

# Exclude build output
output/
dist/
public/

5. Test Configurations

Always test configuration changes:

# Test build before deployment
stattic --output test-build

# Check generated files
ls test-build/

Summary

Stattic's configuration system provides:

  • Flexible configuration via YAML files or command line
  • Environment-specific settings for different deployment stages
  • Simple defaults that work out of the box
  • Powerful customization for advanced use cases

Key Configuration Files

  • stattic.yml - Main configuration file
  • stattic.dev.yml - Development overrides
  • stattic.prod.yml - Production settings

Essential Settings

  • site_url - Required for RSS feeds and sitemaps
  • site_title - Appears in page titles
  • posts_per_page - Controls pagination
  • minify - Optimizes assets for production

Command Line Usage

# Basic build
stattic

# Production build
stattic --minify --robots public

# Development build (no minification)
stattic --robots private

# Custom directories
stattic --content src --output dist

Start with the defaults, then customize as your site grows in complexity. The configuration system is designed to be simple for beginners while powerful enough for advanced use cases.
