Command Line Interface (CLI) Flags

May 20, 2025 · Robert

Each CLI flag lets you customize how Stattic builds your site. These flags can be passed when running stattic.py from the terminal.

--output

Description:
Specifies the directory where the final static site will be generated.

Type:** str
**Default: 'output'

Usage Example:

python3 stattic.py --output public-site

Details:
This flag tells Stattic where to place the generated HTML, CSS, JS, and asset files after the build process. If the directory doesn't exist, it will be created. Useful for keeping multiple versions (e.g. output-dev, output-prod), or aligning with your hosting structure (e.g. public, docs).

--content

Description:
Specifies the path to the folder containing your Markdown content files.

Type:** str
**Default: 'content'

Usage Example:

python3 stattic.py --content ./src/pages

Details:
This directory should include subfolders like /posts/, /pages/, and optionally /authors/, /tags/, and /categories/. Stattic will recursively scan this directory for .md files and parse them into static pages.

--templates

Description:
Sets the path to the Jinja2 templates folder.

Type:** str
**Default: Path is auto-resolved via os.path.join(os.path.dirname(__file__), "templates")

Usage Example:

python3 stattic.py --templates ./theme/templates

Details:
Templates define how each type of content (post, page, index, etc.) is rendered. Customizing this path lets you switch between themes or layouts easily.

--assets

Description:
Defines the path to static assets (CSS, JS, images, etc.) that will be copied into the output directory.

Type:** str
**Default: None (optional)

Usage Example:

python3 stattic.py --assets ./theme/assets

Details:
All files in this directory will be copied as-is to your output folder, preserving the structure. Useful for custom stylesheets, JavaScript files, logos, fonts, and favicon.

--posts-per-page

Description:
Controls how many blog posts appear on each paginated archive page.

Type:** int
**Default: 5

Usage Example:

python3 stattic.py --posts-per-page 10

Details:
Applies only to the blog index page. Controls pagination behavior if you have many blog posts and want to spread them across multiple pages.

--sort-by

Description:
Determines how posts are sorted on blog index pages.

Type:** str
Choices:** 'date', 'title', 'author', 'order' Default: 'date'

Usage Example:

python3 stattic.py --sort-by title

Details:
Defines the ordering of posts on the blog homepage and paginated archive.

  • date: Newest first (standard behavior)
  • title: Alphabetical A--Z
  • author: Groups by author
  • order: Uses a custom order field in your front matter

--fonts

Description:
Specifies Google Fonts to include in your site.

Type:** str
**Default: None

Usage Example:

python3 stattic.py --fonts "Inter,Roboto+Mono"

Details:
Stattic will fetch and include the specified fonts via the Google Fonts API. Separate multiple font families with a comma. You can also include weights like so: "Inter:wght@400;700".

--site-title

Description:
Sets the site's global title, used in templates and metadata.

Type:** str
**Default: "" (empty)

Usage Example:

python3 stattic.py --site-title "Devio Digital"

Details:
Used for the <title> tag, RSS feed, llms.txt, sitemap, and any template that uses the site_title variable.

--site-tagline

Description:
Sets the site's tagline or subtitle.

Type:** str
**Default: "" (empty)

Usage Example:

python3 stattic.py --site-tagline "Build faster, publish better."

Details:
Optional secondary title that appears in metadata and can be injected into templates via the site_tagline variable.

--site-url

Description:
Defines the full public URL of the site for use in absolute links, RSS feeds, sitemap, and social sharing.

Type:** str
**Default: None

Usage Example:

python3 stattic.py --site-url https://example.com

Details:
This should match your deployment URL. It's used to generate full links in RSS, sitemap.xml, llms.txt, and when constructing canonical URLs or Open Graph meta tags.

--robots

Description:
Controls whether robots.txt allows or blocks indexing by search engines.

Type:** str
Choices:** 'public', 'private' Default: 'public'

Usage Example:

python3 stattic.py --robots private

Details:
This flag determines the contents of the generated robots.txt file.

  • public: Allows all crawlers to index the site.
  • private: Blocks all crawlers with Disallow: /.

Useful when you're testing locally or deploying a staging version you don't want indexed.

--watch

Description:
Enables file watching and automatic rebuilding during development.

Type:** bool
Default:** False Usage: Flag only (no value needed)

Usage Example:

python3 stattic.py --watch

Details:
When enabled, Stattic will monitor your content and template directories. Any changes will automatically trigger a rebuild, making it ideal for live development environments.

--minify

Description:
Minifies all CSS and JS files in the final output.

Type:** bool
Default:** False Usage: Flag only (no value needed)

Usage Example:

python3 stattic.py --minify

Details:
Uses csscompressor and rjsmin to reduce file size of stylesheets and scripts. This helps improve performance for production builds. If you're debugging layout or script issues, leave this flag off.

--blog-slug

Description:
Sets the base slug (URL path) for blog post archives.

Type:** str
**Default: 'blog'

Usage Example:

python3 stattic.py --blog-slug posts

Details:
Changes the URL path under which all blog posts are output. For example, using --blog-slug posts will generate posts at /posts/ instead of /blog/. Useful for customizing your site structure or matching legacy URLs.
