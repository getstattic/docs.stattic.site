Front Matter

May 20, 2025 · Robert

Each Markdown file (.md) used in Stattic can contain a YAML front matter block at the top of the file. This metadata controls how the page or post is rendered, where it appears, and how it's displayed in templates.

‚úÖ Basic Format

---
title: "My First Blog Post"
date: "2024-04-01"
slug: "my-first-post"
author: "jdoe"
categories:
 - announcements
tags:
  - launch
  - stattic
template: post
featured_image: /assets/images/header.png
description: "A short summary for SEO and social sharing"
order: 1
draft: false
---

üß† Supported Front Matter Fields

Field Type Required? Description
title string ‚úÖ Yes The display title of the page/post
date string (ISO-8601) ‚úÖ Yes (for posts) Publish date used for sorting and display
slug string ‚ùå Optional Output filename / URL path; auto-generated from title if omitted
author string ‚ùå Optional Author ID used to pull from /content/authors/*.md
categories list of strings ‚ùå Optional Slugs that match /content/categories/
tags list of strings ‚ùå Optional Slugs that match /content/tags/
template string ‚ùå Optional Name of the template to use (e.g., post, page-about)
featured_image string (URL) ‚ùå Optional Used for blog cards and Open Graph tags
description string ‚ùå Optional SEO description and social preview
order integer ‚ùå Optional Used when sorting by --sort-by order
draft boolean ‚ùå Optional If true, the post/page will not be rendered
excerpt string ‚ùå Optional Custom excerpt for preview cards (overrides auto-generation)

üìù Notes

  • Missing date: If no date is provided, the current date is used.
  • Missing slug: Automatically derived from the file name or title.
  • template fallback:
    • If template: post-custom.html exists, it will be used.
    • Falls back to post.html or page.html based on content type.
  • draft: true: Skips rendering but allows version control/testing.
  • Custom Fields: Any extra fields can be accessed in templates using metadata.YOURFIELD.
