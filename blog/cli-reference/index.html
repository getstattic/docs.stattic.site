Stattic CLI Reference

June 19, 2025 · Robert

Complete command-line interface reference for Stattic. This guide covers all available commands, options, and practical usage examples to help you master the Stattic CLI.

Overview

Stattic provides a powerful command-line interface for building static sites. The CLI allows you to:

  • Build sites with custom configurations
  • Override configuration file settings
  • Initialize new projects
  • Customize directory structures
  • Optimize build processes

Basic Syntax

stattic [OPTIONS]

All options are optional - running stattic without arguments will build your site using default settings or your stattic.yml configuration file.

Basic Usage

Build Your Site

# Build with default settings
stattic

# Build with configuration file
stattic  # Uses stattic.yml if present

# Build with custom output directory
stattic --output public

# Build with minification enabled
stattic --minify

Get Help and Version

# Show help message
stattic --help

# Show version information
stattic --version

Command Reference

Complete Options List

Based on stattic --help, here are all available options:

Option Type Description Example
-h, --help Flag Show help message and exit stattic --help
--output OUTPUT String Output directory for generated site --output public
--content CONTENT String Content directory containing markdown files --content src
--templates TEMPLATES String Templates directory --templates themes
--assets ASSETS String Assets directory to copy to output --assets static
--posts-per-page POSTS_PER_PAGE Integer Number of posts per page for pagination --posts-per-page 5
--sort-by {date,title,author,order} Choice Sort posts by field --sort-by title
--fonts FONTS String Comma-separated list of Google Fonts --fonts "Inter,Roboto"
--site-title SITE_TITLE String Site title for metadata --site-title "My Blog"
--site-tagline SITE_TAGLINE String Site tagline for metadata --site-tagline "Web Dev Blog"
--site-url SITE_URL String Site URL for RSS feeds and sitemaps --site-url https://example.com
--robots {public,private} Choice Robots.txt configuration --robots private
--llms {public,private} Choice LLMs.txt configuration for AI crawlers --llms private
--watch Flag Watch for file changes and rebuild (not implemented) --watch
--minify Flag Minify CSS and JS assets --minify
--blog-slug BLOG_SLUG String Custom slug for posts instead of 'blog' --blog-slug posts
--init {yml,yaml,json} Choice Create a sample configuration file --init yml
--version Flag Show program's version number and exit --version

Directory Options

Output Directory

Control where your generated site is saved:

# Default output directory
stattic  # Uses 'output' directory

# Custom output directory
stattic --output public
stattic --output dist
stattic --output build
stattic --output ../deploy

Use Cases:

  • --output public - GitHub Pages, Netlify
  • --output dist - Modern web development convention
  • --output build - Create React App style
  • --output ../deploy - Deploy to parent directory

Content Directory

Specify where your Markdown files are located:

# Default content directory
stattic  # Uses 'content' directory

# Custom content directory
stattic --content src
stattic --content posts
stattic --content docs
stattic --content ../content-repo

Use Cases:

  • --content src - Source-based project structure
  • --content posts - Simple blog structure
  • --content docs - Documentation site
  • --content ../content-repo - Separate content repository

Templates Directory

Override default templates:

# Use custom templates
stattic --templates themes/custom
stattic --templates layouts
stattic --templates ../shared-templates

Use Cases:

  • --templates themes/custom - Custom theme development
  • --templates layouts - Alternative naming convention
  • --templates ../shared-templates - Shared across projects

Assets Directory

Control which assets are copied to output:

# Default assets
stattic --assets assets

# Custom assets directory
stattic --assets static
stattic --assets public
stattic --assets resources

Use Cases:

  • --assets static - Common static file convention
  • --assets public - Public assets directory
  • --assets resources - Resource-based naming

Content Configuration

Posts Per Page

Control pagination for your site:

# Default pagination (5 posts per page)
stattic

# Custom pagination
stattic --posts-per-page 5   # Default value
stattic --posts-per-page 10  # More posts per page
stattic --posts-per-page 20  # Many posts per page

Recommendations:

  • Blogs: 5-10 posts per page
  • News sites: 15-20 posts per page
  • Documentation: 50+ items per page
  • Portfolio: 6-12 items per page

Post Sorting

Control how posts are ordered:

# Sort by date (newest first)
stattic --sort-by date

# Sort alphabetically by title
stattic --sort-by title

# Sort by author name
stattic --sort-by author

# Sort by manual order (frontmatter 'order' field)
stattic --sort-by order

Use Cases:

  • --sort-by date - Blogs, news sites
  • --sort-by title - Documentation, references
  • --sort-by author - Multi-author sites
  • --sort-by order - Curated content, tutorials

Blog URL Structure

Customize your blog URL structure:

# Default blog URLs (/blog/post-name/)
stattic

# Custom blog slug
stattic --blog-slug posts    # /posts/post-name/
stattic --blog-slug articles # /articles/post-name/
stattic --blog-slug news     # /news/post-name/
stattic --blog-slug ""       # /post-name/ (root level)

Site Metadata

Site Title

Set your site's title:

# Set site title
stattic --site-title "My Personal Blog"
stattic --site-title "Company Documentation"
stattic --site-title "Portfolio Site"

Appears in:

  • Browser tab titles
  • RSS feed titles
  • Social media shares
  • Search engine results

Site Tagline

Add a descriptive tagline:

# Set site tagline
stattic --site-tagline "Thoughts on web development"
stattic --site-tagline "Complete API documentation"
stattic --site-tagline "Full-stack developer portfolio"

Appears in:

  • Site headers
  • RSS feed descriptions
  • Meta descriptions
  • Social media shares

Site URL

Set your site's canonical URL:

# Set site URL (required for RSS feeds and sitemaps)
stattic --site-url https://example.com
stattic --site-url https://blog.company.com
stattic --site-url https://username.github.io

Required for:

  • RSS feed generation
  • XML sitemap creation
  • Canonical link tags
  • Social media metadata

SEO and Crawlers

Robots.txt Configuration

Control search engine crawling:

# Allow all search engines (default)
stattic --robots public

# Block all search engines
stattic --robots private

Generated robots.txt:

Public:

User-agent: *
Allow: /
Sitemap: https://your-site.com/sitemap.xml

Private:

User-agent: *
Disallow: /

LLMs.txt for AI Crawlers

Control AI/LLM crawler access:

# Allow AI crawlers (default)
stattic --llms public

# Block AI crawlers
stattic --llms private

Use Cases:

  • --llms public - Open to AI training, content indexing
  • --llms private - Protect content from AI scraping

Build Options

Minification

Optimize CSS and JavaScript assets:

# Enable minification
stattic --minify

# Default (no minification)
stattic

Minification benefits:

  • Smaller file sizes
  • Faster page loads
  • Reduced bandwidth usage
  • Better performance scores

Development vs Production:

# Development build (readable assets)
stattic

# Production build (optimized assets)
stattic --minify

Google Fonts

Include Google Fonts in your site:

# Single font
stattic --fonts "Inter"

# Multiple fonts (comma-separated)
stattic --fonts "Inter,Roboto"
stattic --fonts "Poppins,Source Code Pro"
stattic --fonts "Playfair Display,Inter,JetBrains Mono"

Font Selection Tips:

  • Body text: Inter, Roboto, Open Sans
  • Headings: Poppins, Montserrat, Playfair Display
  • Code: Source Code Pro, JetBrains Mono, Fira Code
  • Limit to 2-3 fonts for performance

Watch Mode

# Enable watch mode (not yet implemented)
stattic --watch

Note: Watch mode is listed in the help but not yet implemented in the current version.

Initialization

Create Configuration Files and Starter Structure

Generate sample configuration files and complete project structure:

# Create YAML configuration and complete starter structure (recommended)
stattic --init yml

# Create YAML configuration (alternative)
stattic --init yaml

# Create JSON configuration
stattic --init json

Generated files:

  • --init yml → stattic.yml + complete project structure
  • --init yaml → stattic.yaml + complete project structure
  • --init json → stattic.json + complete project structure

Complete starter structure created:

your-project/
├── stattic.yml              # Configuration file
├── templates/               # Template files
│   ├── base.html
│   ├── index.html
│   ├── post.html
│   ├── page.html
│   ├── tag.html
│   ├── category.html
│   └── 404.html
├── content/                 # Content directories
│   ├── posts/
│   │   └── sample-post.md   # Sample blog post
│   ├── pages/
│   │   └── sample-page.md   # Sample page
│   ├── authors.yml          # Author definitions
│   ├── categories.yml       # Category definitions
│   └── tags.yml            # Tag definitions
└── assets/                  # Asset files
    ├── css/
    │   ├── tailwind.min.css
    │   └── stattic.css
    ├── js/
    │   └── alpine.min.js
    └── images/              # For your images

Sample generated configuration:

# Stattic Configuration File
# Configure your static site generator settings here

# Site information
site_url: https://example.com
site_title: My Static Site
site_tagline: Built with Stattic

# Build settings
output: output
content: content
templates: templates
assets: assets
blog_slug: blog

# Content settings
posts_per_page: 5
sort_by: date  # date, title, author, order

# Styling
fonts:
  - Quicksand
  - Open Sans

# SEO settings

# Development settings

watch: false

Common Workflows

Development Workflow

# Initial setup
stattic --init yml

# Development builds (fast, readable)
stattic --robots private

# Test different configurations
stattic --posts-per-page 5 --sort-by title

# Preview with different fonts
stattic --fonts "Poppins,Source Code Pro"

Production Workflow

# Production build (optimized)
stattic --minify --robots public --site-url https://example.com

# Deploy to specific directory
stattic --output ../deploy/public --minify

# Multi-step deployment
stattic --minify --output dist
# Then deploy dist/ directory

Content Management

# Organize content differently
stattic --content src/posts --assets src/static

# Change URL structure
stattic --blog-slug articles --posts-per-page 8

# Sort content for different audiences
stattic --sort-by order  # Curated order
stattic --sort-by title  # Alphabetical

Multi-Environment Setup

# Local development
stattic --site-url http://localhost:8000 --robots private

# Staging environment
stattic --site-url https://staging.example.com --robots private --minify

# Production environment
stattic --site-url https://example.com --robots public --minify

Advanced Usage

Combining Options

# Complete production build
stattic \
  --output public \
  --minify \
  --robots public \
  --llms public \
  --site-title "My Professional Blog" \
  --site-tagline "Web development insights" \
  --site-url https://myblog.com \
  --posts-per-page 8 \
  --sort-by date \
  --fonts "Inter,Source Code Pro"

# Documentation site build
stattic \
  --content docs \
  --output build \
  --blog-slug guides \
  --posts-per-page 50 \
  --sort-by order \
  --site-title "Project Documentation" \
  --minify

# Portfolio site build
stattic \
  --content projects \
  --blog-slug work \
  --posts-per-page 12 \
  --sort-by order \
  --fonts "Playfair Display,Inter" \
  --minify

Configuration File Override

Override specific settings from your stattic.yml:

# Config file has minify: false, but enable for production
stattic --minify

# Override site URL for staging
stattic --site-url https://staging.example.com

# Test different pagination
stattic --posts-per-page 5

# Try different fonts
stattic --fonts "Roboto,JetBrains Mono"

Directory Structure Flexibility

# Monorepo structure
stattic \
  --content website/content \
  --assets website/static \
  --templates website/themes \
  --output website/build

# Separate content repository
stattic \
  --content ../content-repo \
  --assets ../content-repo/images \

stattic \
  --content src/markdown \
  --assets src/public \

Troubleshooting

Common Issues and Solutions

1. Build Fails

Issue: Command fails with directory errors

# Check if directories exist
ls -la content/
ls -la assets/

# Create missing directories
mkdir -p content assets output

Solution: Ensure all specified directories exist

2. No Output Generated

Issue: Command runs but no files in output directory

# Verify content exists
ls -la content/
find content/ -name "*.md"

# Check for errors
stattic --help  # Verify command syntax

Solution: Ensure content directory has .md files

3. Fonts Not Loading

Issue: Google Fonts not appearing on site

# Check font names (case-sensitive)
stattic --fonts "Inter"      # ✅ Correct
stattic --fonts "inter"      # ❌ Wrong case
stattic --fonts "Inter Font" # ❌ Wrong name

Solution: Use exact Google Fonts names

4. Site URL Issues

Issue: RSS feeds or sitemaps not generating

# Ensure complete URL with protocol
stattic --site-url https://example.com  # ✅ Correct
stattic --site-url example.com          # ❌ Missing protocol
stattic --site-url localhost:8000       # ❌ Missing protocol

Solution: Always include https:// or http://

Debugging Commands

    ls -la content/ assets/ templates/

  2. Verify content files

    find content/ -name "*.md" | head -5

  3. Test basic build

    stattic --output test-build
ls -la test-build/

  4. Check for configuration conflicts

    # Remove config file temporarily
mv stattic.yml stattic.yml.backup
stattic --site-title "Test"

Best Practices

Development Best Practices

1. Use Configuration Files

# ✅ Good: Use config file for consistent settings
# Create stattic.yml with your standard settings
stattic  # Uses config file

# ❌ Avoid: Long command lines for regular builds
stattic --site-title "Blog" --minify --robots public --fonts "Inter"

2. Environment-Specific Commands

# Development
stattic --robots private

# Staging
stattic --robots private --minify

# Production
stattic --robots public --minify --site-url https://example.com

3. Use Meaningful Directory Names

# ✅ Clear directory names
stattic --output public --content posts --assets static

# ❌ Confusing names
stattic --output x --content y --assets z

Performance Best Practices

1. Enable Minification for Production

# Development (fast builds)
stattic

# Production (optimized)
stattic --minify

2. Optimize Font Loading

# ✅ Limit fonts for performance
stattic --fonts "Inter,Source Code Pro"

# ❌ Too many fonts
stattic --fonts "Inter,Roboto,Poppins,Playfair Display,JetBrains Mono"

3. Configure Appropriate Pagination

# Blog sites
stattic --posts-per-page 10

# Portfolio sites
stattic --posts-per-page 12

# Documentation sites
stattic --posts-per-page 50
