System Requirements

Minimum Requirements

  • Python: 3.8 or higher
  • Operating System: Windows 10+, macOS 10.14+, or Linux (any modern distribution)
  • Memory: 512MB RAM available
  • Storage: 100MB free disk space (plus space for your sites)
  • Internet: Required for package installation

Recommended Requirements

  • Python: 3.10 or higher
  • Memory: 1GB RAM or more
  • Storage: 500MB+ free disk space
  • Git: For version control and deployment

Dependencies Included

Stattic comes with all necessary dependencies:

  • Jinja2: Template engine
  • Markdown: Content processing
  • Pillow: Image processing
  • PyYAML: Configuration files
  • Watchdog: File watching (for future watch mode)
  • Requests: HTTP requests
  • Tqdm: Progress bars
  • CSS/JS minifiers: Asset optimization

Installation Methods

Method 1: From PyPI (Recommended)

Stattic is available as a PyPI package. Install it using pip:

# Install Stattic
pip install stattic

# Verify installation
stattic --version

Method 2: Using Virtual Environment (Recommended for Development)

# Create a virtual environment
python -m venv stattic-env

# Activate the virtual environment
# On Windows:
stattic-env\Scripts\activate
# On macOS/Linux:
source stattic-env/bin/activate

# Install Stattic
pip install stattic

# Verify installation
stattic --version

Method 3: From Source (For Development)

If you want to contribute to Stattic or use the latest development version:

# Clone the repository
git clone https://github.com/getstattic/stattic.git
cd stattic

# Create a virtual environment (recommended)
python -m venv venv

# Activate the virtual environment
# On Windows:
venv\Scripts\activate
# On macOS/Linux:
source venv/bin/activate

# Install in development mode
pip install -e .

# Verify installation
stattic --version

Platform-Specific Setup

Windows Setup

Option 1: Using Windows Subsystem for Linux (WSL)

# Install WSL2 (if not already installed)
wsl --install

# Install Python in WSL
sudo apt update
sudo apt install python3 python3-pip python3-venv

# Install Stattic
pip3 install stattic

# Verify installation
stattic --version

Option 2: Native Windows Installation

# Check Python installation
python --version

# Install Stattic
pip install stattic

# Verify installation
stattic --version

# If command not found, ensure Python Scripts are in PATH
# Usually: C:\Users\YourUsername\AppData\Local\Programs\Python\Python3x\Scripts

macOS Setup

Using pip (Recommended)

# If you have Python 3.8+ installed
pip3 install stattic

# Verify installation
stattic --version

Using Homebrew Python

# Install Python via Homebrew (if needed)
brew install python

# Install Stattic
pip3 install stattic

# Verify installation
stattic --version

Linux Setup

Ubuntu/Debian

# Update package list
sudo apt update

# Install Python and pip (if not already installed)
sudo apt install python3 python3-pip python3-venv

# Install Stattic
pip3 install stattic

# Add to PATH if needed
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# Verify installation
stattic --version

CentOS/RHEL/Fedora

# Install Python and pip
sudo dnf install python3 python3-pip

# Install Stattic
pip3 install stattic

# Verify installation
stattic --version

Arch Linux

# Install Python (usually pre-installed)
sudo pacman -S python python-pip

# Install Stattic
pip install stattic

# Verify installation
stattic --version

Creating Your First Project

After installing Stattic, create your first static site:

Initialize a New Project

# Create a new directory for your project
mkdir my-stattic-site
cd my-stattic-site

# Initialize with YAML configuration (recommended)
stattic --init yml

# Or initialize with JSON configuration
stattic --init json

Available CLI Options

Based on the actual CLI, here are the available options:

# Basic site generation
stattic

# Custom output directory
stattic --output ./public

# Custom content directory
stattic --content ./my-content

# Custom templates directory
stattic --templates ./my-templates

# Custom assets directory
stattic --assets ./my-assets

# Site configuration
stattic --site-title "My Site" --site-tagline "My awesome site"

# Posts configuration
stattic --posts-per-page 5 --sort-by date

# Custom blog slug
stattic --blog-slug articles

# Asset optimization
stattic --minify

# SEO and crawlers
stattic --robots public --llms public

# Include Google Fonts
stattic --fonts "Inter,Roboto"

# Initialize new project
stattic --init yml    # Creates YAML config
stattic --init json   # Creates JSON config

# Check version
stattic --version

Troubleshooting

Common Installation Issues

Issue: "command not found: stattic"

Solutions:

# Check if Stattic was installed correctly
pip show stattic

# Check if Python Scripts directory is in PATH
python -m site --user-base
# Add {user-base}/bin (Linux/macOS) or {user-base}\Scripts (Windows) to PATH

# Reinstall with --user flag
pip install --user stattic

# Or use Python module directly
python -m stattic_pkg.cli --version

Issue: Permission denied during installation

Solutions:

# Install to user directory (recommended)
pip install --user stattic

# Or use virtual environment
python -m venv venv
source venv/bin/activate  # Linux/macOS
# venv\Scripts\activate     # Windows
pip install stattic

Issue: Python version compatibility

Solutions:

# Check Python version (must be 3.8+)
python --version

# If using older Python, upgrade or use pyenv
pyenv install 3.10.0
pyenv local 3.10.0
pip install stattic

Issue: SSL certificate errors

Solutions:

# Upgrade certificates
pip install --upgrade certifi

# Use trusted hosts (temporary)
pip install --trusted-host pypi.org --trusted-host pypi.python.org --trusted-host files.pythonhosted.org stattic

Issue: "No module named 'stattic_pkg'"

Solutions:

# Ensure proper installation
pip uninstall stattic
pip install stattic

# Check if in virtual environment
which python
which pip

# Try reinstalling dependencies
pip install --force-reinstall stattic

Platform-Specific Issues

Windows Issues

  • PATH issues: Add %APPDATA%\Python\Python3x\Scripts to PATH
  • PowerShell execution policy: Run Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
  • Long path names: Enable long path support in Windows settings

macOS Issues

  • Command Line Tools: Install with xcode-select --install
  • Permission issues: Use --user flag or virtual environments
  • Python not found: Ensure Python 3.8+ is installed (use python3 command)

Linux Issues

  • Missing build tools: Install build-essential package (Ubuntu/Debian)
  • Python development headers: Install python3-dev package
  • Permission issues: Avoid using sudo with pip (use --user flag instead)

Build Issues

Issue: "No such file or directory" during build

Solutions:

# Ensure you're in the correct directory
ls -la  # Should show stattic.yml

# Check configuration file
cat stattic.yml

# Verify directory structure
ls content/ templates/ assets/

Issue: Templates not found

Solutions:

# Check templates directory exists
ls templates/

# Verify required templates
ls templates/base.html templates/index.html

# Re-initialize if templates are missing
stattic --init yml

Issue: Assets not copied

Solutions:

# Check assets directory
ls assets/

# Verify CSS files exist
ls assets/css/

# Ensure assets path is correct in config
grep assets stattic.yml

CI/CD Setup

GitHub Actions Example

# .github/workflows/build.yml
name: Build Stattic Site

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v4

    - name: Set up Python
      uses: actions/setup-python@v4
      with:
        python-version: '3.10'

    - name: Install Stattic
      run: |
        python -m pip install --upgrade pip
        pip install stattic

    - name: Build site
      run: stattic

    - name: Test build
      run: |
        test -f output/index.html
        test -f output/sitemap.xml
        echo "‚úÖ Build successful"

Environment Variables

While Stattic doesn't currently support environment variables directly, you can use them in scripts:

# Set environment variables
export STATTIC_SITE_TITLE="My Blog"
export STATTIC_OUTPUT_DIR="./public"

# Use in build script
stattic --site-title "$STATTIC_SITE_TITLE" --output "$STATTIC_OUTPUT_DIR"

for version in 3.8.18 3.9.18 3.10.12 3.11.5; do

  pyenv local $version
  pip install stattic
  stattic --version
done
