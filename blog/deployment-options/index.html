Deployment options

June 19, 2025 · Robert

Complete guide to deploying your Stattic-generated static sites across popular hosting platforms. Since Stattic generates standard HTML/CSS/JS files, you have complete freedom to choose any static hosting provider.

Overview

Stattic generates completely static websites that can be deployed anywhere that serves HTML files. This decentralized approach gives you complete control over your hosting choice based on your needs:

  • Free options: GitHub Pages, Netlify, Cloudflare Pages
  • Performance-focused: Cloudflare Pages, AWS CloudFront
  • Developer-friendly: Vercel, Netlify, Firebase
  • Enterprise: AWS, Azure, Google Cloud
  • Self-hosted: Your own server, VPS, or dedicated hosting

Pre-Deployment Checklist

Before deploying, ensure your site is production-ready:

# Generate your site
stattic

# Test locally (use any static server)
python -m http.server 8000 -d output

# Check output directory
ls -la output/

Essential Checks

  • All pages render correctly in local preview
  • Images and assets load properly
  • Internal links work without broken references
  • SEO meta tags are present in HTML
  • 404 page exists (output/404.html)
  • Sitemap generated (output/sitemap.xml)
  • Robots.txt configured (output/robots.txt)

GitHub Pages

Free hosting directly from your GitHub repository.

Setup Steps

  1. Create GitHub Repository

    git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/username/your-site.git
git push -u origin main

  2. Configure GitHub Pages

    • Go to repository Settings â†’ Pages
    • Source: "Deploy from a branch"
    • Branch: main (or gh-pages)
    • Folder: / (root) or /docs

  3. Deploy Options

    Option A: Manual Upload

    # Build your site
stattic

# Copy output to repository root (or docs/ folder)
cp -r output/* ./
git add .
git commit -m "Deploy site"
git push

    Option B: GitHub Actions (Recommended)

    # .github/workflows/deploy.yml
name: Deploy Stattic Site

on:
  push:
    branches: [main]
  workflow_dispatch:

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Setup Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.9'

      - name: Install Stattic
        run: pip install stattic

      - name: Build site
        run: stattic

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./output

GitHub Pages Specifics

  • Custom domain: Configure in repository settings
  • HTTPS: Automatically enabled
  • Build time: ~1-2 minutes
  • Limitations: 1GB bandwidth/month, 100GB storage

Cloudflare Pages

Fast, global CDN with excellent performance and generous free tier.

Setup Steps

  1. Connect Repository

    • Visit Cloudflare Pages
    • Connect your GitHub/GitLab repository
    • Or use direct upload for manual deployment

  2. Build Configuration

    # Build settings in Cloudflare dashboard
Build command: pip install stattic && stattic
Build output directory: output
Root directory: /
Environment variables:
  PYTHON_VERSION: 3.9

  3. Manual Deployment

    # Build your site
stattic

# Install Wrangler CLI
npm install -g wrangler

# Deploy
wrangler pages publish output --project-name=your-site

Cloudflare Pages Features

  • Global CDN: 200+ edge locations
  • Custom domains: Free SSL certificates
  • Analytics: Built-in web analytics
  • Edge functions: Serverless functions at the edge
  • Preview deployments: Every git push gets a preview URL

Netlify

Developer-focused platform with powerful features and intuitive interface.

Setup Steps

  1. Connect Repository

    • Visit Netlify
    • "New site from Git" â†’ Connect repository

  2. Build Settings

    # netlify.toml (optional)
[build]
  command = "pip install stattic && stattic"
  publish = "output"

[build.environment]
  PYTHON_VERSION = "3.9"

[[redirects]]
  from = "/*"
  to = "/404.html"
  status = 404

  3. Manual Deployment

    # Build your site
stattic

# Install Netlify CLI
npm install -g netlify-cli

# Deploy
netlify deploy --prod --dir=output

Netlify Features

  • Form handling: Process form submissions
  • Edge functions: Serverless functions
  • Split testing: A/B testing built-in
  • Asset optimization: Automatic image optimization
  • Deploy previews: Every PR gets a preview URL

Firebase Hosting

Google's hosting platform with excellent integration to Firebase services.

Setup Steps

  1. Install Firebase CLI

    npm install -g firebase-tools
firebase login

  2. Initialize Project

    firebase init hosting

# Configuration:
# Public directory: output
# Single-page app: No
# Automatic builds: Optional

  3. Deploy

    # Build your site
stattic

# Deploy to Firebase
firebase deploy

Firebase Configuration

// firebase.json
{
  "hosting": {
    "public": "output",
    "ignore": [
      "firebase.json",

AWS S3 + CloudFront

Enterprise-grade hosting with AWS's global infrastructure.

Setup Steps

  1. Create S3 Bucket

    # Install AWS CLI
pip install awscli
aws configure

# Create bucket
aws s3 mb s3://your-site-bucket

  2. Configure Static Website Hosting

    # Enable static website hosting
aws s3 website s3://your-site-bucket \
  --index-document index.html \
  --error-document 404.html

  3. Deploy Site

    # Build your site
stattic

# Upload to S3
aws s3 sync output/ s3://your-site-bucket --delete

  4. Set up CloudFront (Optional)

    • Create CloudFront distribution
    • Point to S3 bucket
    • Configure custom domain and SSL

AWS Benefits

  • Scalability: Handle any traffic level
  • Global CDN: CloudFront edge locations
  • Custom domains: Route 53 integration
  • Advanced features: Lambda@Edge, security headers

Self-Hosted Options

Host on your own infrastructure for complete control.

Nginx Configuration

# /etc/nginx/sites-available/your-site
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/your-site;
    index index.html;

    # Handle clean URLs
    location / {
        try_files $uri $uri/ =404;
    }

    # Custom 404 page
    error_page 404 /404.html;

    # Security headers
    add_header X-Content-Type-Options nosniff;
    add_header X-Frame-Options DENY;
    add_header X-XSS-Protection "1; mode=block";
}

Apache Configuration

# .htaccess in output directory
RewriteEngine On

# Custom 404 page
ErrorDocument 404 /404.html

# Security headers
Header always set X-Content-Type-Options nosniff
Header always set X-Frame-Options DENY
Header always set X-XSS-Protection "1; mode=block"

# Cache static assets
<IfModule mod_expires.c>
    ExpiresActive On
    ExpiresByType text/css "access plus 1 year"
    ExpiresByType image/png "access plus 1 year"
    ExpiresByType image/jpg "access plus 1 year"
    ExpiresByType image/jpeg "access plus 1 year"
    ExpiresByType image/gif "access plus 1 year"
    ExpiresByType application/javascript "access plus 1 year"
</IfModule>

Docker Deployment

# Dockerfile
FROM nginx:alpine

# Copy static files
COPY output/ /usr/share/nginx/html/

# Custom nginx config (optional)
COPY nginx.conf /etc/nginx/nginx.conf

EXPOSE 80

# Build and run
docker build -t my-stattic-site .
docker run -p 80:80 my-stattic-site

CI/CD Automation

Automate your deployment process with continuous integration.

GitHub Actions Example

# .github/workflows/deploy.yml
name: Build and Deploy

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v3

    - name: Set up Python
      uses: actions/setup-python@v4
      with:
        python-version: '3.9'

    - name: Install dependencies
      run: |
        pip install stattic

    - name: Build site
      run: stattic

    - name: Upload build artifacts
      uses: actions/upload-artifact@v3
      with:
        name: site-build
        path: output/

  deploy:
    needs: build
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'

    steps:
    - name: Download build artifacts
      uses: actions/download-artifact@v3
      with:
        name: site-build
        path: output/

    - name: Deploy to Netlify
      uses: nwtgck/actions-netlify@v1.2
      with:
        publish-dir: './output'
        production-branch: main
      env:
        NETLIFY_AUTH_TOKEN: ${{ secrets.NETLIFY_AUTH_TOKEN }}
        NETLIFY_SITE_ID: ${{ secrets.NETLIFY_SITE_ID }}

GitLab CI Example

# .gitlab-ci.yml
stages:
  - build
  - deploy

build:
  stage: build
  image: python:3.9
  script:
    - pip install stattic
    - stattic
  artifacts:
    paths:
      - output/
    expire_in: 1 hour

deploy:
  stage: deploy
  image: alpine:latest
  script:
    - apk add --no-cache curl
    - curl -X POST -d {} $DEPLOY_WEBHOOK
  only:
    - main

Domain Configuration

Custom Domain Setup

  1. DNS Configuration

    # A Record (for root domain)
Type: A
Name: @
Value: [hosting provider IP]

# CNAME (for www subdomain)
Type: CNAME
Name: www
Value: your-site.netlify.app

  2. SSL Certificate

    • Most platforms provide free SSL automatically
    • Custom certificates can be uploaded if needed

Subdomain Setup

# For blog.yourdomain.com
Type: CNAME
Name: blog
Value: your-site.netlify.app

Performance Optimization

Build Optimization

# Enable compression during build
stattic --minify

# Check available options
stattic --help

CDN Configuration

Most platforms include CDN automatically, but you can optimize:

  1. Cache Headers

    Static assets: Cache for 1 year
HTML files: Cache for 1 hour
API responses: Cache for 5 minutes

  2. Compression

    • Enable Gzip/Brotli compression
    • Most platforms enable this automatically

Monitoring

Set up monitoring for:

  • Uptime: UptimeRobot, Pingdom
  • Performance: Google PageSpeed Insights
  • Analytics: Google Analytics, Plausible
  • Error tracking: Sentry (for JavaScript errors)

Troubleshooting

Common Issues

Issue Cause Solution
404 on navigation Missing clean URL support Configure redirects/rewrites
Assets not loading Incorrect base URL Check asset paths in templates
Build failures Missing dependencies Verify build commands
SSL errors Certificate issues Check domain configuration
Slow loading No CDN/optimization Enable compression and CDN

Conclusion

Stattic's static output gives you complete freedom to choose the hosting platform that best fits your needs. Whether you prefer the simplicity of GitHub Pages, the performance of Cloudflare, or the control of self-hosting, your Stattic site will run beautifully anywhere.

The key is to:

  1. Build locally first and test thoroughly
  2. Choose the right platform for your requirements
  3. Set up automation for seamless deployments
  4. Monitor performance and optimize as needed
