Quick start guide

June 20, 2025 · Robert

Get up and running with Stattic in just a few minutes! This guide will take you from installation to your first deployed static site.

What is Stattic?

Stattic is a fast, lightweight static site generator written in Python. It transforms your Markdown content into beautiful, optimized static websites with features like:

  • 🚀 Lightning fast builds and page loads
  • 📱 Mobile-responsive themes with Tailwind CSS and Alpine.js
  • 🖼️ Automatic image optimization (WebP conversion)
  • 🎨 Google Fonts integration
  • 📊 SEO-friendly with automatic sitemaps, robots.txt, and llms.txt
  • � RSS feeds for blog content
  • ⚡ Asset minification for CSS and JavaScript

Step 1: Installation

Prerequisites

  • Python 3.8 or higher
  • pip (Python package installer)

Install Stattic

pip install stattic

Verify Installation

stattic --version

Step 2: Create Your First Project

Initialize a New Project

# Create a new directory for your site
mkdir my-awesome-site
cd my-awesome-site

# Initialize Stattic project with YAML config
stattic --init yml

# Or initialize with JSON config
stattic --init json

This creates the following structure:

my-awesome-site/
├── content/
│   ├── posts/               # Blog posts go here
│   │   └── sample-post.md   # Sample blog post
│   ├── pages/               # Static pages go here
│   │   └── sample-page.md   # Sample about page
│   ├── authors.yml          # Author information
│   ├── categories.yml       # Post categories
│   └── tags.yml             # Post tags
├── templates/               # Jinja2 templates
│   ├── base.html            # Base template
│   ├── index.html           # Homepage
│   ├── post.html            # Blog post template
│   ├── page.html            # Static page template
│   ├── category.html        # Category listing
│   ├── tag.html             # Tag listing
│   └── 404.html             # Error page
├── assets/                  # CSS, JS, images
│   ├── css/
│   │   ├── tailwind.min.css # Tailwind CSS framework
│   │   └── stattic.css      # Custom styles
│   ├── js/
│   │   └── alpine.min.js    # Alpine.js framework
│   └── images/              # Image directory
└── stattic.yml              # Configuration file

Step 3: Create Your First Content

Write Your First Blog Post

Create content/posts/hello-world.md:

---
title: "Hello, World!"
custom_url: hello-world
author: 1
date: 2025-06-20
categories:
  - 1
tags:
  - 1
description: "My first post with Stattic!"
keywords: "hello world, first post, stattic"
---

# Welcome to My New Site!

This is my first post using **Stattic**. I'm excited to start blogging!

## What I Love About Stattic

- Easy to use Markdown
- Beautiful themes with Tailwind CSS
- Fast performance
- Great SEO features
- Automatic image optimization

Check out more features in future posts!

Step 4: Configure Your Site

Edit stattic.yml to customize your site:

# Site information
site_url: https://your-domain.com
site_title: My Awesome Site
site_tagline: Sharing knowledge, one post at a time

# Build settings
output: output
content: content
templates: templates
assets: assets
blog_slug: blog

# Content settings
posts_per_page: 5
sort_by: date  # date, title, author, order

# Styling
fonts:
  - Quicksand      # Available Google Fonts
  - Open Sans

# SEO settings
robots: public  # public or private

# Development settings
minify: false
watch: false

Step 5: Build Your Site

Build the Site

stattic

This will:

  • Load configuration from stattic.yml
  • Process all Markdown files in content/
  • Apply Jinja2 templates
  • Optimize images (convert to WebP)
  • Generate sitemap.xml
  • Create robots.txt and llms.txt
  • Generate RSS feed
  • Copy assets to output directory
  • Minify CSS/JS (if enabled)

You'll see output like:

Loaded configuration from: stattic.yml
Building index page
Building 404 page
Generating RSS feed
Generating XML sitemap
Generating robots.txt
Generating llms.txt
Site build completed in 1.30 seconds.
Total posts generated: 1
Total pages generated: 1
Total images converted to WebP: 0

View Your Site

Your built site will be in the output/ directory:

output/
├── index.html              # Homepage
├── 404.html               # Error page
├── sitemap.xml            # XML sitemap
├── robots.txt             # Robots configuration
├── llms.txt               # AI crawler configuration
├── blog/                  # Blog posts
│   └── hello-world/
│       └── index.html
├── about/                 # Static pages
│   └── index.html
├── feed/                  # RSS feed
│   └── index.xml
├── assets/                # Copied assets
└── images/                # Optimized images

To view your site locally:

# Using Python's built-in server
cd output
python -m http.server 8000

# Visit http://localhost:8000

Step 6: Customize Your Theme

Update Site Styling

The default theme uses Tailwind CSS and Alpine.js. You can customize it by editing:

  • assets/css/stattic.css - Custom styles
  • templates/base.html - Layout and structure
  • Individual templates for specific page types

Add Your Logo

Place your logo in assets/images/logo.png and update the header in templates/base.html.

Customize Navigation

Edit the pages front matter to control navigation:

---
title: "Contact"
order: 2         # Navigation order (pages only)
---

Step 7: Deploy Your Site

Option 1: GitHub Pages

  1. Push your code to GitHub
  2. In repository settings, enable GitHub Pages
  3. Set source to "GitHub Actions"
  4. Create .github/workflows/build.yml:
name: Build and Deploy
on:
  push:
    branches: [ main ]
jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4
    - name: Setup Python
      uses: actions/setup-python@v4
      with:
        python-version: '3.10'
    - name: Install dependencies
      run: pip install stattic
    - name: Build site
      run: stattic
    - name: Deploy to GitHub Pages
      uses: peaceiris/actions-gh-pages@v3
      with:
        github_token: ${{ secrets.GITHUB_TOKEN }}
        publish_dir: ./output

Option 2: Netlify

  1. Connect your GitHub repository to Netlify
  2. Set build command: pip install stattic && stattic
  3. Set publish directory: output
  4. Deploy!

Option 3: Traditional Hosting

Upload the entire output/ folder to your web server via FTP/SFTP.

Common Commands

# Initialize new project with YAML config
stattic --init yml

# Initialize new project with JSON config  
stattic --init json

# Build site with default settings
stattic

# Build with custom output directory
stattic --output ./public

# Build with custom content directory
stattic --content ./my-content

# Build with minification enabled
stattic --minify

# Build with custom site settings
stattic --site-title "My Site" --site-tagline "My tagline"

# Build with custom fonts
stattic --fonts "Inter,Roboto"

# Build with custom posts per page
stattic --posts-per-page 10 --sort-by title

# Get help
stattic --help

# Check version
stattic --version

Troubleshooting

Build Errors

  • "No module named 'stattic'": Install with pip install stattic
  • "No such file or directory": Ensure you're in the project directory with stattic.yml
  • "Template not found": Ensure templates exist in templates/ directory or re-run stattic --init yml

Content Issues

  • Posts not showing: Check front matter format and ensure files are in content/posts/
  • Pages not generating: Verify front matter and file location in content/pages/
  • Images not optimizing: Check file formats (JPG/PNG supported for WebP conversion)

Performance Issues

  • Large sites: Enable minification with --minify flag
  • Slow builds: Check for very large images or reduce image sizes
