Content creation

June 19, 2025 · Robert

Master the art of creating compelling content for your Stattic site. This guide covers everything from basic Markdown syntax to advanced content organization, helping you create engaging blogs, documentation, and websites.

Overview

Stattic uses Markdown files** with **YAML front matter for all content. This combination provides:

  • Easy writing experience with Markdown syntax
  • Rich metadata through YAML front matter
  • Flexible organization with directory structure
  • SEO optimization through structured data

Content File Structure

Every content file follows this pattern:

---
title: "Your Title Here"
date: "2024-01-15"
# ... other metadata
---

Front Matter Essentials

Front matter is YAML metadata at the top of each file, enclosed by triple dashes (---).

Required Front Matter

---
title: "Your Post Title"        # Required: Page/post title  
date: "2025-06-20"             # Required for posts: Publication date (YYYY-MM-DD)
---

Common Front Matter Fields

---
# Essential fields
title: "Complete Guide to Static Sites"
date: "2025-06-20"
description: "Learn how to build amazing static sites"

# Organization (use numeric IDs from YAML files)
categories: [1, 2]             # List of category IDs from categories.yml
tags: [1, 3, 5]               # List of tag IDs from tags.yml
author: 1                      # Author ID from authors.yml

# SEO and metadata
keywords: "static sites, web development, tutorial"
featured_image: "images/static-sites-hero.jpg"

# URL customization
custom_url: "guides/static-sites"

# Template selection
template: "tutorial"           # Uses post-tutorial.html template

# Language
lang: "en"                     # Language code (default: en)
---

Front Matter Reference

Field Type Description Example
title String Page/post title "My Blog Post"
date String Publication date (YYYY-MM-DD) "2025-06-20"
description String Meta description for SEO "Learn about..."
categories List Category IDs from categories.yml [1, 2]
tags List Tag IDs from tags.yml [1, 3, 5]
author Number Author ID from authors.yml 1
featured_image String Featured image path "images/hero.jpg"
custom_url String Custom permalink "custom-path"
keywords String SEO keywords "seo, web"
template String Custom template name "tutorial"
lang String Language code "en"

Performance Optimization

Content Performance

Optimize for Speed

# ‚úÖ Performance-friendly content
- Use compressed images
- Write concise, scannable content
- Minimize external embeds
- Use code blocks instead of screenshots when possible

# ‚ùå Performance killers
- Huge unoptimized images
- Excessive external widgets
- Auto-playing videos
- Bloated content with unnecessary details

Content Length Guidelines

# Optimal lengths by content type
- Blog posts: 1,500-2,500 words
- Tutorials: 2,000-4,000 words
- Quick tips: 500-800 words
- About pages: 300-600 words
- Product descriptions: 150-300 words

3. Image Problems

# ‚ùå Common issues
- Broken image paths (not relative to assets/images/)
- Missing alt text
- Huge file sizes
- Unsupported image formats

# ‚úÖ Solutions
- Use paths relative to assets/images/: `images/my-photo.jpg`
- Always include descriptive alt text
- Compress images before uploading (under 500KB recommended)
- Use standard formats: JPEG, PNG, WebP, SVG
